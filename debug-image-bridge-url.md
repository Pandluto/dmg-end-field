# [OPEN] image-bridge-url

## 现象
- 图像列表与路径表看起来已对齐，但图片仍不显示。
- 怀疑桥接 URL 或显示层仍有问题。

## 当前目标
- 验证 `/user-images/*` 桥接 URL 是否真的能返回图片。
- 找出是桥接服务、端口、路径映射还是前端显示层的问题。

## 初始假设
- H1: 前端请求的桥接端口与 Electron 实际监听端口不一致，导致 URL 根本没打到正确服务。
- H2: 前端生成的 `/user-images/...` 相对路径与桥接服务解析规则不一致，导致 404。
- H3: 桥接服务能返回文件，但页面实际使用了另一套 URL 构造逻辑，导致显示层没有走桥接。
- H4: 当前激活版本目录里文件存在，但路径表中的 `canonicalPath/publicUrl` 仍有旧值，导致拿错地址。
- H5: 桥接服务本身未启动或被其他服务占用端口，导致图片请求失败。

## 验证计划
- 读取主进程和前端的桥接常量与 URL 构造代码。
- 确认 Electron 实际监听端口。
- 确认前端 `img src` 使用的 origin 和 path。
- 如需要，启动应用后直接请求一个真实 `/user-images/...` URL 验证返回。
