## 1. Bug：拖拽技能按钮时不跟随鼠标

### 当前问题

从沙盒拖拽技能按钮时，按钮没有跟随鼠标移动

### 原因分析

全局 mousemove 事件监听器中的 `skillButtons` 状态可能没有及时更新，导致找不到对应的按钮

### 修改方案

```typescript
useEffect(() => {
  if (!globalDragState) return;

  const handleMouseMove = (e: MouseEvent) => {
    // 使用 ref 存储最新的 skillButtons，而不是依赖闭包中的 state
    // 或者直接使用 globalDragState 中存储的初始位置
  };
}, [globalDragState]);
```

## 2. 技术问题：按钮应跟随鼠标直到点击画布

### 当前问题

点击沙盒按钮后，按钮位置没有立即跟随鼠标

### 期望行为

```
1. mousedown on 沙盒按钮
   → 创建技能按钮实例
   → 按钮位置 = 当前鼠标位置
   → 按钮立即跟随鼠标

2. mousemove (全局)
   → 按钮位置持续跟随鼠标
   → Y轴锁定在对应谱线上

3. mouseup on 画布
   → 吸附到最近的节点
   → 完成放置
```

### 修改方案

关键点：在创建按钮时，初始位置应该设为当前鼠标位置，而不是谱线起点

```typescript
const handleSandboxDragStart = (e) => {
  // 初始位置 = 当前鼠标位置
  const initialX = e.clientX;
  const initialY = e.clientY;

  // 但 Y 轴需要锁定在对应谱线上
  const lineY = calculateLineY(config, staffIndex, lineIndex) + staffHeight / 2;

  const newButton = {
    position: { x: initialX, y: lineY },
    // ...
  };
};
```

## 3. 设计优化：节点间隔和按钮同步缩小一倍

### 当前问题

节点间隔和按钮尺寸过大

### 修改方案

| 元素 | 当前值 | 修改后 |
| --- | --- | --- |
| 节点间距 | ≈44px | **≈22px** |
| 技能按钮直径 | 88px | **44px** |
| 谱线间距 | ≥100px | **不变** |

### 实现

```typescript
// types/index.ts
export const DEFAULT_CANVAS_CONFIG: CanvasConfig = {
  // ... 其他配置
  skillButtonSize: 44,  // 88 → 44
  nodeSpacing: 22,      // 新增：节点间距
};
```

## 4. 其他修复

### 4.1 Canvas 坐标转换

按钮位置使用屏幕坐标，但渲染时需要相对于画布的坐标

```typescript
// 需要将屏幕坐标转换为画布坐标
const canvasRect = canvasRef.current.getBoundingClientRect();
const canvasX = e.clientX - canvasRect.left;
const canvasY = e.clientY - canvasRect.top;
```

### 4.2 全局拖拽状态管理

使用 ref 来存储 skillButtons，避免闭包问题

```typescript
const skillButtonsRef = useRef(skillButtons);
skillButtonsRef.current = skillButtons;

useEffect(() => {
  const handleMouseMove = (e: MouseEvent) => {
    const buttons = skillButtonsRef.current;
    // 使用 buttons 而不是 skillButtons
  };
}, []);
```
