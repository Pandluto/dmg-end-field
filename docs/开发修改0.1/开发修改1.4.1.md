# 开发修改 1.4.1 - 谱线组间距bug修复

## 约束
- 本计划仅针对1.4.0的组间距bug修复
- 不修改技能按钮拖拽逻辑
- 不修改谱线组数控制按钮逻辑
- 保持现有视觉风格

## 问题分析

### bug表现
点击增加谱线组数按钮时，同一组四线谱之间的间距被压缩，但锚点/节点位置没有相应调整。

### 根因
当前 `calculateLineY` 基于固定 `canvasHeight` 计算所有线的位置，当增加组数时：
- 每条线的 Y 坐标通过 `(canvasHeight - margins - groupSpacing) / (totalLines - 1)` 计算
- 增加 staffCount 后，总线数增加，但 canvasHeight 没变
- 导致 lineSpacing 变小，组内4条线挤在一起

### 渲染结构问题
当前所有谱线渲染在单一 `.canvas` 容器中，使用 `top: xxxpx` 绝对定位。当增加组数时，布局重新计算导致现有线位置被压缩。

## 技术方案

### 方案：独立谱线组容器
每组四线谱使用独立容器，容器高度固定为单组所需高度。

### 渲染结构
```
.canvas-container
  overflow-y: auto
  .staff-group (每组一个容器)
    .staff-line x 4
    .damage-node x (4 x nodeCount)
    .staff-label x 4
  .staff-group x staffCount
```

### 配置变更
```typescript
// 新增
staffGroupHeight: number;  // 单组容器高度，默认300
```

### calculateLineY 修改
```typescript
// calculateLineY 现在只计算单组内的相对位置
// 不再依赖 staffCount 和 groupSpacing
export function calculateLineY(
  config: CanvasConfig,
  lineIndex: number  // 0-3
): number {
  const lineSpacing = (config.staffGroupHeight - config.staffMarginTop - config.staffMarginBottom - 4 * config.staffHeight) / 3;
  return config.staffMarginTop + lineIndex * (lineSpacing + config.staffHeight);
}
```

### 组件修改
```typescript
// CanvasBoard/index.tsx
// 1. renderLines 改为 renderStaffGroups
// 2. 每组渲染一个容器，容器内渲染4条线
// 3. staffCount 变化时，只改变容器数量，不重新计算单组内布局
```

## 实现步骤

1. 修改 `types/index.ts` - 添加 `staffGroupHeight` 配置
2. 修改 `utils/layout.ts`:
   - `calculateLineY` 只接收 `lineIndex` 参数，计算单组内相对位置
   - 更新 `getLineSpacing` 适配新逻辑
3. 修改 `CanvasBoard/index.tsx`:
   - 添加 `renderStaffGroup` 函数，渲染单组容器
   - 修改 `renderLines` 调用，循环渲染 `staffCount` 个组
   - 确保组内间距不随组数变化
4. 测试验证
