## 背景

第一版代码存在以下理解错误，需要纠正后重新实现。

## ❌ 理解错误 1：画布形式

### 错误理解

每行是一条四线谱。

### 正确理解

- 横向多行模式
- **每行都有 4 条线（为一组）**
- 4 条线对应 4 名干员

### 示意图（正确）

```
每行（4条线为一组）：
[管理员谱线] ════════════════════════════════════════════════
[汤汤谱线]   ════════════════════════════════════════════════
[洛茜谱线]   ════════════════════════════════════════════════
[干员4谱线]  ════════════════════════════════════════════════
            ●  ●  ●  ●  ●  ●  ●  ●  ●  ●  ← 伤害节点
```

## ❌ 理解错误 2：技能按钮

### 错误理解

- 技能按钮直径是 40px
- 每名干员只有固定的 4 个技能按钮（每个技能类型一个）
- 技能按钮一开始就显示在谱线上

### 正确理解

- 技能按钮直径 = **3 个间隔**，正好覆盖 4 个节点
- **技能按钮是无限个的**
- 技能按钮由**屏幕右侧沙盒**里面点击拖拽出来
- 如同**棋子**一样操作

### 示意图

```
┌─────────────────────────────────────────────────────────────────────┐
│  右侧沙盒 (技能按钮库)                                                │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                                    │
│  │  A  │ │  B  │ │  E  │ │  Q  │  ← 点击拖拽出来                   │
│  └─────┘ └─────┘ └─────┘ └─────┘                                    │
│                                                                     │
│  可以无限拖拽出技能按钮，放到谱线上                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## ❌ 理解错误 3：谱线标识

### 错误理解

每行谱线前方标注干员头像。

### 正确理解

- 第一行四线谱的**第 1 根谱线** = 管理员谱线，标识 "管理员"
- 第一行四线谱的**第 2 根谱线** = 汤汤谱线，标识 "汤汤"
- 第一行四线谱的**第 3 根谱线** = 洛茜谱线，标识 "洛茜"
- 第一行四线谱的**第 4 根谱线** = 第4名干员谱线
- **第二行的四线谱同理**

## 谱线吸附规则（保持）

- 技能按钮拖拽到谱线上后，**Y轴固定**在对应谱线
- **X轴自由移动**：在谱线上左右移动
- **吸附规则**：技能按钮的**边缘**与伤害节点对齐
- **禁止重叠**：同一谱线上的技能按钮不能重叠
