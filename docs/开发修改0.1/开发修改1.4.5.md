# 开发修改 1.4.5 - 节点占满时禁止吸附

## 约束
- 本计划仅针对1.4.4的节点占满时重叠bug修复
- 不修改谱线组数控制逻辑
- 不修改拖拽核心逻辑
- 新增提示弹窗功能

## 问题分析

### Bug: 节点占满时按钮重叠

**表现：**
当某条谱线的所有40个节点都被占满时，再拖动新按钮到这条线上会重叠到最后一个节点上。

**根因：**
`snapToNearestNode` 的防重叠逻辑：
```typescript
for (let offset = 0; offset <= config.nodeCount; offset++) {
  // 检查 offset 为 0, 1, 2, ... 39 时的情况
  // 如果所有节点都被占用，循环结束后 targetNodeIndex 停留在最后一个被检查的节点（可能是被占用的）
}
```

当所有节点都被占用时，for循环会检查所有节点，但所有都被占用，所以不会进入 if 分支，targetNodeIndex 停留在原始计算的 nearestNodeIndex（可能被占用的节点）。

## 技术方案

### 方案：返回可用性标志

修改 `snapToNearestNode` 返回值，添加 `isAvailable` 标志：

```typescript
return {
  snappedPosition: { x: snappedX, y: lineY + groupOffset },
  nodeIndex: targetNodeIndex,
  isAvailable: true,  // 新增：是否找到可用节点
};
```

在防重叠检查逻辑中，如果找到可用节点设置 `isAvailable = true`，否则保持 `false`。

### CanvasBoard 处理

在吸附时检查 `isAvailable`：
- 如果为 `false`，弹出提示"该谱线节点已满"
- 不创建新按钮，撤销操作

## 实现步骤

1. 修改 `layout.ts` - `snapToNearestNode` 返回值添加 `isAvailable` 标志
2. 修改 `CanvasBoard/index.tsx` - 检测 `isAvailable`，为 false 时弹窗提示并撤销
3. 测试验证节点占满时不能吸附
