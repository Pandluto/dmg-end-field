## 问题分析

### Bug 1: 最后一组第4条谱线溢出画布

**表现：**
- 2组时，第2组第4条线在画布外
- 3组时，第3组第4条线在画布外
- 以此类推

**根因分析：**
`getTotalCanvasHeight` 计算逻辑问题。当前公式：
```
groupHeight = lineCount * (lineSpacing + staffHeight)
totalHeight = staffCount * groupHeight + (staffCount - 1) * groupSpacing
```

但 `calculateLineY` 的公式使用 `staffGroupHeight`，导致：
- `groupHeight` 实际占用 = `staffGroupHeight` = 300px
- `lineSpacing` = (300 - marginTop - marginBottom - 4*4) / 3

当画布高度 = 2*300 + 80 = 680px，但第2组实际需要 top = 300 + 80 = 380px，第4条线在 380 + calculateLineY(3) ≈ 380 + 267 ≈ 647px，接近680px边界可能溢出。

**真正根因：**
`renderStaffGroup` 的 `top: groupOffset` 使用 `getGroupOffset`，但画布总高度计算时，组的偏移计算与实际渲染不一致。

`getGroupOffset` 公式：
```
groupOffset = staffIndex * (lineCount * (lineSpacing + staffHeight) + groupSpacing)
```

这里 `lineSpacing` 使用的是 `getLineSpacing(config)`，而 `getLineSpacing` 也是基于 `staffGroupHeight` 计算的。问题在于 `groupHeight` 在两处计算应该一致，但实际不一致。

### Bug 2: 干员标签与谱线Y轴不居中对齐

**表现：**
标签没有和对应谱线在Y轴上居中对齐。

**根因：**
`staff-label` 的 `top` 使用 `y - 20`，但谱线高度是4px，中心是 `y + 2`。标签应该垂直居中于线中心。

**正确计算：**
```
label top = lineY + staffHeight/2 - labelHeight/2
         = (y + 2) - 10
         = y - 8
```
或直接使用 `y + staffHeight/2 - 10`。

## 技术方案

### Bug 1 修复
检查 `getGroupOffset` 和 `getTotalCanvasHeight` 的计算一致性。确保：
- 每组容器高度 = `staffGroupHeight`
- 组间偏移正确
- 画布总高度足够容纳所有组

可能需要调整 `staffGroupHeight` 或 `groupSpacing`，确保：
```
canvasHeight >= staffCount * staffGroupHeight + (staffCount - 1) * groupSpacing
```

### Bug 2 修复
```typescript
// 标签位置 = 线中心 - 标签高度一半
const labelTop = y + config.staffHeight / 2 - 10;  // 假设标签高20px
```
