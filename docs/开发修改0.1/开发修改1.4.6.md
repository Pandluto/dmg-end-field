# 开发修改 1.4.6 - 防止右键长按时选中文字

## 约束
- 本计划仅针对交互优化，防止用户操作时选中文字
- 不修改功能逻辑
- 仅在技能按钮组件上添加 mousedown 事件阻止默认行为
- 阻止行为仅影响按钮区域，不影响页面其他可交互元素

## 问题分析

### Bug: 右键长按时选中文字

**表现：**
在技能按钮上右键长按时，会触发浏览器默认的文本选择行为，导致按钮文字被选中。

**根因：**
浏览器默认允许在元素上选中文字，mousedown 事件触发后开始选区创建。

## 技术方案

### JS 方案：mousedown preventDefault()

```typescript
const handleMouseDown = (e: React.MouseEvent) => {
  e.preventDefault();
};

<button onMouseDown={handleMouseDown} className="skill-button">
  {skill.name}
</button>
```

**原理：**
- `mousedown` 是浏览器选中文本流程的起点
- 在源头阻止默认行为，切断选中文本的触发链路
- 不依赖 CSS，兼容性更好

**副作用可控：**
- 仅在按钮组件上阻止，不影响页面其他交互
- 按钮本身不需要默认行为（按钮是自定义交互）

## 实现步骤

1. 修改 `SkillButton.tsx` - 添加 `onMouseDown` 事件处理，调用 `e.preventDefault()`
2. 测试验证右键长按不选中文字
3. 测试验证左键双击不选中文字
