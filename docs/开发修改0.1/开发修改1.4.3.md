## 问题分析

### Bug 1: 技能按钮可重叠吸附

**表现：**
同一个干员的多个技能按钮可以吸附到同一个节点上，完全重合。

**根因：**
`snapToNearestNode` 没有检查目标节点是否已被占用。

**解决方案：**
在吸附时检查目标节点是否已被该干员的按钮占用，如被占用则拒绝吸附或吸附到最近可用节点。

### Bug 2: 第41、42节点溢出谱线

**表现：**
每条谱线的第41、42个节点没有被谱线覆盖，溢出到外面了。

**根因：**
谱线使用 `left: 80px; right: 40px;`，但节点位置计算：
```
nodeX = marginLeft + nodeIndex * nodeSpacing
```
当 nodeIndex=40, nodeSpacing=22 时：
```
nodeX = 80 + 40 * 22 = 80 + 880 = 960px
```

但画布宽度约 960px，减去 `marginRight=40`，谱线实际宽度只有 920px（80到920）。节点960超出了谱线范围。

**解决方案：**
确保谱线宽度能覆盖所有节点：
```
staffLineWidth = marginLeft + (nodeCount - 1) * nodeSpacing + nodeSize/2
              = 80 + 39 * 22 + 8 = 80 + 858 + 8 = 946px
```
需要让谱线 `right` 为 0，或者调整 `nodeCount`。

## 技术方案

### Bug 1 修复

```typescript
// 吸附前检查
function isNodeOccupied(
  staffIndex: number,
  lineIndex: number,
  nodeIndex: number,
  skillButtons: SkillButton[],
  characterId: string
): boolean {
  return skillButtons.some(
    btn =>
      btn.characterId === characterId &&
      btn.staffIndex === staffIndex &&
      btn.lineIndex === lineIndex &&
      getNodeIndexFromPosition(btn.position) === nodeIndex
  );
}
```

修改吸附逻辑：
- 优先吸附到目标节点
- 如果被占用，尝试附近节点
- 如果最近节点都被占用，拒绝吸附

### Bug 2 修复

修改 CSS，让谱线宽度覆盖所有节点：

```css
.staff-line {
  left: 80px;
  right: 0;  /* 改为0，让谱线延伸到容器边缘 */
}
```

或调整 `nodeCount` 减少节点数。
