## 待修改内容

## 1. 修改1：按钮直径

### 当前问题

技能按钮直径 = 3 个节点间隔 ≈ 132px

### 修改方案

技能按钮直径 = **2 个节点间隔** ≈ 88px

---

## 2. 功能疏漏：技能按钮右键删除

### 当前问题

从沙盒拖拽到画布的技能按钮无法删除

### 修改方案

- 技能按钮添加**右键菜单**或直接右键删除
- 监听 `onContextMenu` 事件
- 右键点击按钮时，弹出确认删除选项
- 或者：直接右键删除，无需确认

---

## 3. 设计缺陷：谱线间隔应大于按钮直径

### 当前问题

谱线之间的间距可能小于按钮直径，导致按钮重叠

### 修改方案

- 谱线间距 ≥ 按钮直径 + 一定余量
- 确保同一谱线上的技能按钮不会与其他谱线的按钮重叠

---

## 4. 设计优化：四线谱留两组就可以

### 当前问题

目前有 4 组谱线（每组 4 条线 = 16 条线），可能过多

### 修改方案

- 保留 **2 组谱线**（每组 4 条线 = 8 条线）
- 每组对应不同的两名干员的攻击顺序（待定）

---

## 5. 技术问题：拖拽吸附逻辑

### 当前问题

从沙盒拖拽技能按钮时，按钮位置不正确

### 修改方案

拖拽流程修改为：

```
1. mousedown on 沙盒技能按钮
   → 创建技能按钮实例
   → 按钮位置 = 当前鼠标位置
   → 设置 isDragging = true

2. mousemove (全局)
   → 更新按钮位置跟随鼠标
   → 按钮 Y 轴锁定在对应的谱线上
   → X 轴跟随鼠标

3. mouseup on 画布
   → 吸附到最近的节点
   → 设置 isDragging = false
   → 完成放置
```

### 关键点

- 全局监听 mousemove 事件
- Y 轴始终固定在对应谱线上
- 松开时吸附到最近的伤害节点

## 6. 实现错误：按钮只能吸附到对应谱线

### 当前问题

管理员的技能按钮可以吸附到汤汤的谱线上

### 修改方案

每个技能按钮只能吸附到**对应干员的谱线**：

- 管理员的 A/B/E/Q 按钮 → 只吸附到管理员的谱线
- 汤汤的 A/B/E/Q 按钮 → 只吸附到汤汤的谱线

### 实现方式

```typescript
// 拖拽时锁定 Y 轴位置
const targetLineY = calculateLineY(config, staffIndex, lineIndex);

// staffIndex 和 lineIndex 由创建按钮时决定
// 管理员: staffIndex=0, lineIndex=0
// 汤汤: staffIndex=0, lineIndex=1
// 洛茜: staffIndex=0, lineIndex=2
// ...
```
