# 开发修改 0.2.6 - 技能图标底色适配 + 画布按钮图标渲染

## 约束

- 本计划修改两件事：① colorMap 底色适配；② 画布技能按钮图标渲染
- 不修改拖拽算法、节点吸附逻辑、战斗计算逻辑
- 不修改头像路径规则（头像路径不变）
- 资源文件名严格遵循已有规范，**禁止臆造**
- 每个 `<img>` 必须配备 `onError` 回退策略

---

## 问题分析

### 问题一：colorMap 底色饱和度偏高

当前 `getElementBackgroundColor` 的颜色饱和度偏高，与半透明 PNG 贴图叠加后视觉偏深。需要调低饱和度，提升清爽感。

| element | 当前色值 | 建议色值 |
|---|---|---|
| `physical` | `rgba(210,210,210,0.75)` 淡灰 | `rgba(220,220,225,0.70)` 更浅灰 |
| `ice` | `rgba(180,220,255,0.75)` 冰蓝 | `rgba(200,235,255,0.70)` 更浅冰蓝 |
| `fire` | `rgba(255,180,140,0.75)` 火红 | `rgba(255,210,195,0.70)` 更浅暖 |
| `electric` | `rgba(230,230,120,0.75)` 电气黄 | `rgba(245,245,190,0.70)` 更浅黄 |
| `ether` | `rgba(200,160,255,0.75)` 异核紫 | `rgba(215,195,255,0.70)` 更浅紫 |
| `nature` | `rgba(120,200,140,0.75)` 自然绿 | `rgba(175,225,185,0.70)` 更浅绿 |

### 问题二：画布技能按钮图片消失 Bug

**表现**：技能沙盒中拖拽到画布并吸附后，技能图标 `<img>` 消失，只剩文字。

**根因分析**：

画布技能按钮由 `SkillButtonComponent`（`SkillButton.tsx`）渲染，渲染逻辑如下：

```tsx
// SkillButton.tsx（当前代码）
<div className="canvas-skill-button">
  <span className="skill-label">{skillType}</span>       {/* 永远渲染文字 */}
  <span className="skill-character-name">{characterName}</span>
</div>
```

`SkillButton` 接口（`types/index.ts`）仅包含：

```typescript
// types/index.ts - 当前 SkillButton 字段
export interface SkillButton {
  id: string;
  characterId: string;
  characterName: string;
  skillType: SkillType;
  position: Position;
  staffIndex: number;
  lineIndex: number;
  isDragging: boolean;
  isSelected: boolean;
  isFromSandbox: boolean;
  // ❌ 缺少 skillIconUrl、avatarUrl
}
```

`useCanvasDrag.ts` 在 `handleMouseUp` 中创建新按钮时：

```typescript
// useCanvasDrag.ts - 创建新按钮（当前代码）
const newButton: SkillButton = {
  id: draggingState.id,
  characterId: draggingState.characterId,
  characterName: draggingState.characterName,
  skillType: draggingState.skillType,
  position: snappedPosition,
  staffIndex, lineIndex,
  isDragging: false,
  isSelected: false,
  isFromSandbox: true,
  // ❌ 没有传入 skillIconUrl
};
```

**结论**：`SkillButton` 接口缺少 `skillIconUrl`，`SkillButtonComponent` 无法渲染图标，导致永远只能显示文字兜底。

---

## 技术方案

### 第一步：扩展 SkillButton 类型，添加 skillIconUrl

修改 `src/types/index.ts`，在 `SkillButton` 接口增加可选字段：

```typescript
export interface SkillButton {
  // ... 既有字段不变 ...

  // 【新增】技能图标 URL（由 useCanvasDrag 创建按钮时从 Character 派生）
  skillIconUrl?: string;
}
```

### 第二步：在 useCanvasDrag 中创建按钮时注入 skillIconUrl

修改 `src/components/CanvasBoard/hooks/useCanvasDrag.ts`：

- 从 `draggingState` 解构出 `characterId`
- 从 `characters`（或 `skillButtonsRef` 中已有点的 characterId 反查）中找到对应角色的 `avatarUrl`
- 通过 `resolveSkillIconUrl(characterName, skillType)` 拼接图标路径
- 注入到 `newButton.skillIconUrl`

注入逻辑示例：

```typescript
// 获取角色对应的技能图标 URL（从全局 characters 列表或通过 characterId 反查）
const skillIconUrl = resolveSkillIconUrl(draggingState.characterName, draggingState.skillType);

const newButton: SkillButton = {
  id: draggingState.id,
  characterId: draggingState.characterId,
  characterName: draggingState.characterName,
  skillType: draggingState.skillType,
  position: snappedPosition,
  staffIndex,
  lineIndex,
  isDragging: false,
  isSelected: false,
  isFromSandbox: true,
  skillIconUrl,  // 【新增】
};
```

### 第三步：改造 SkillButtonComponent，渲染技能图标

修改 `src/components/CanvasBoard/SkillButton.tsx`：

- 优先渲染 `<img className="skill-icon">`
- 图片加载失败时退回 `skillType` 文字
- 保持圆形按钮布局不变（`border-radius: 50%`）

关键代码结构：

```tsx
export function SkillButtonComponent({ button, size, onMouseDown, onContextMenu }: SkillButtonProps) {
  const { position, skillType, isSelected, isDragging, characterName, skillIconUrl } = button;

  return (
    <div
      className={`canvas-skill-button ${isSelected ? 'selected' : ''} ${isDragging ? 'dragging' : ''}`}
      style={{
        left: position.x - size / 2,
        top: position.y - size / 2,
        width: size,
        height: size,
        backgroundColor: SKILL_COLORS[skillType],
      }}
      onMouseDown={handleMouseDown}
      onContextMenu={onContextMenu}
    >
      {skillIconUrl ? (
        <img
          className="skill-icon"
          src={skillIconUrl}
          alt={SKILL_LABELS[skillType]}
          onError={(e) => {
            // 图片加载失败时隐藏 img，文字 span 仍显示
            (e.target as HTMLImageElement).style.display = 'none';
          }}
        />
      ) : null}
      <span className="skill-label">{skillType}</span>
      <span className="skill-character-name">{characterName}</span>
    </div>
  );
}
```

### 第四步：调整 colorMap 半透明色值

修改 `src/utils/assetResolver.ts` 的 `getElementBackgroundColor`：

```typescript
const colorMap: Record<string, string> = {
  physical:  'rgba(220, 220, 225, 0.70)',   // 更浅灰
  ice:       'rgba(200, 235, 255, 0.70)',   // 更浅冰蓝
  fire:      'rgba(255, 210, 195, 0.70)',   // 更浅暖
  electric:  'rgba(245, 245, 190, 0.70)',   // 更浅黄
  ether:     'rgba(215, 195, 255, 0.70)',   // 更浅紫
  nature:    'rgba(175, 225, 185, 0.70)',   // 更浅绿
};
```

### 第五步：适配 SkillButtonComponent 的 CSS

修改 `src/components/CanvasBoard/SkillButton.css`：

```css
.canvas-skill-button {
  position: relative;
  overflow: hidden;
}

.canvas-skill-button .skill-icon {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  border-radius: 50%;
  pointer-events: none;
}

.canvas-skill-button .skill-label {
  position: relative;
  z-index: 1;
}

.canvas-skill-button .skill-character-name {
  position: relative;
  z-index: 1;
}
```

## 防幻觉约束（显式声明）

| 约束项 | 说明 |
|---|---|
| 资源路径 | 严格使用 `resolveSkillIconUrl(characterName, skillType)` 拼接，不自行发明 |
