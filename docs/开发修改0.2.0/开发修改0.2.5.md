# 开发修改 0.2.5 - 修复技能图标路径错误

## 约束

- 本计划**仅修复 `resolveSkillIconUrl` 的路径拼接逻辑**
- 不修改拖拽算法、节点吸附逻辑、战斗计算逻辑
- 不修改头像路径规则（头像路径 `管理员.png` 不变）
- 资源文件名严格遵循已有规范，**禁止臆造**

---

## 问题分析

### Bug 描述

**当前路径（错误）**

`resolveSkillIconUrl` 当前生成的路径为：

```
/assets/avatars/管理员/skills/A.png
/assets/avatars/管理员/skills/B.png
/assets/avatars/管理员/skills/E.png
/assets/avatars/管理员/skills/Q.png
```

**实际资源路径（正确）**

实际文件名是**中文技能名称**，直接拼接在角色名之后：

```
/assets/avatars/管理员/管理员普攻.png     → 对应 SkillType A
/assets/avatars/管理员/管理员战技.png     → 对应 SkillType B
/assets/avatars/管理员/管理员连携技.png   → 对应 SkillType E
/assets/avatars/管理员/管理员终结技.png   → 对应 SkillType Q
```

**根因**

`resolveSkillIconUrl` 使用了 `skills/<skillType>.png` 子目录结构，但实际资源**没有** `skills/` 子目录，而是直接放在 `public/assets/avatars/<角色名>/` 下，文件名格式为 `<角色名><技能名>.png`。

### SkillType ↔ 技能名映射（来源 `types/index.ts`）

```typescript
export const SKILL_NAMES: Record<SkillType, string> = {
  A: '普通攻击',   // 简称"普攻"
  B: '战技',
  E: '连携技',
  Q: '终结技',
};
```

---

## 技术方案

### 第一步：修正 `resolveSkillIconUrl` 路径拼接规则

修改 `src/utils/assetResolver.ts`，**删除** `skills/` 子目录，改为直接拼接在角色名之后：

**修改前（错误）**

```typescript
export function resolveSkillIconUrl(characterName: string, skillType: SkillType): string {
  return `/assets/avatars/${encodeURIComponent(characterName)}/skills/${skillType}.png`;
}
```

**修改后（正确）**

```typescript
/**
 * 技能图标资源路径解析
 * 规范路径: /assets/avatars/<characterName>/<characterName><skillName>.png
 * 技能类型 A/B/E/Q 对应 普通攻击/战技/连携技/终结技
 * @param characterName - 角色名称
 * @param skillType     - A | B | E | Q
 */
export function resolveSkillIconUrl(characterName: string, skillType: SkillType): string {
  const skillName = SKILL_NAMES[skillType];
  return `/assets/avatars/${encodeURIComponent(characterName)}/${encodeURIComponent(characterName)}${skillName}.png`;
}
```

### 第二步：导入 `SKILL_NAMES`

`resolveSkillIconUrl` 需要引用 `SKILL_NAMES` 映射，因此需在 `assetResolver.ts` 中新增 import：

```typescript
import type { SkillType } from '../types';
import { SKILL_NAMES } from '../types';
```
