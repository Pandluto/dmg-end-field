# 开发修改 0.2.10 - 谱线干员标识改为名称+头像

## 约束

- 本计划仅修改谱线左侧干员标识的渲染方式
- 将纯文字标签改为干员名称+头像的视觉样式
- 不修改拖拽、吸附、谱线渲染等核心逻辑
- 资源路径严格遵循规范，**禁止臆造**

---

## 需求分析

### 当前状态

谱线左侧的干员标识（`CanvasArea.tsx` 第 78-89 行）仅显示纯文字：

```tsx
<div
  className="staff-label"
  style={{ top: y + config.staffHeight / 2 - 10, left: 10 }}
>
  {character?.name || `干员${lineIndex + 1}`}
</div>
```

渲染效果示例：
```
[class="staff-label" style="top: 52px; left: 10px;">管理员]
```

### 目标状态

将谱线左侧的干员标识改为"干员名称 + 头像"样式，与技能沙盒中的 `sandbox-character-header` 保持一致：

```
┌─────────────────────────┐
│ 管理员 [头像图片]        │
└─────────────────────────┘
```

参考 `SkillSandbox` 中的实现：

```tsx
<div className="sandbox-character-header"> {/* flex */}
  <span className="sandbox-character-name">管理员</span>
  <img
    className="sandbox-avatar"
    src="/assets/avatars/管理员/管理员.png"
    alt="管理员 头像"
    style={{ backgroundColor: rgb(80, 80, 85) }}
  />
</div>
```

### 关键差异

| 属性 | 当前 staff-label | 目标 sandbox-character-header |
|------|-----------------|------------------------------|
| 布局 | 绝对定位，独立元素 | flex 布局，干员名+头像并排 |
| 头像 | 无 | 有（圆形，36×36） |
| 背景色 | 无 | 元素对应的半透明底色 |
| 高度 | 仅文字高度 | 需与谱线对齐 |

---

## 技术方案

### 实现思路

由于谱线标签是在画布内绝对定位的（与 `SkillSandbox` 的 flex 布局不同），需要：

1. **新建样式类** `.staff-label-container` 作为包装容器（绝对定位）
2. **复用现有类** `.sandbox-character-name` 和 `.sandbox-avatar`（保持与沙盒一致的外观）
3. **调整布局**：标签容器内使用 flex 横向排列干员名和头像

### 样式映射关系

| SkillSandbox 类名 | 谱线标签新类名 | 用途 |
|------------------|--------------|------|
| `.sandbox-character-header` | `.staff-label-container` | 包装容器（flex） |
| `.sandbox-character-name` | 复用 | 干员名称文字 |
| `.sandbox-avatar` | 复用 | 头像图片（36×36 圆形） |

### 头像资源路径

头像路径已在 `AppContext.tsx` 中通过 `resolveAvatarUrl(character.name)` 注入到 `character.avatarUrl`，直接使用即可。

### 头像背景色

使用 `getElementBackgroundColor(character.element)` 获取元素对应的半透明底色，与 `SkillSandbox` 保持一致。

## 实现步骤

### 步骤 1：修改 `src/components/CanvasBoard/CanvasBoard.css`

1. 新增 `.staff-label-container` 样式（绝对定位 + flex 横向排列 + 透明背景）

```css
/* 谱线左侧干员标识容器（包装名称+头像） */
.staff-label-container {
  position: absolute;
  display: flex;
  align-items: center;
  gap: 6px;
  background: transparent;
  pointer-events: none;
}
```

2. 复用 `.sandbox-character-name` 和 `.sandbox-avatar` 样式（已有，无需新增）

3. 可选：调整 `.staff-label` 样式（如果不再使用可删除）

### 步骤 2：修改 `src/components/CanvasBoard/components/CanvasArea.tsx`

1. 引入 `getElementBackgroundColor`（如果尚未引入）

2. 修改谱线标签渲染逻辑（第 78-89 行）：

将：
```tsx
lines.push(
  <div
    key={`label-${staffIndex}-${lineIndex}`}
    className="staff-label"
    style={{
      top: y + config.staffHeight / 2 - 10,
      left: 10,
    }}
  >
    {character?.name || `干员${lineIndex + 1}`}
  </div>
);
```

改为：
```tsx
lines.push(
  <div
    key={`label-${staffIndex}-${lineIndex}`}
    className="staff-label-container"
    style={{
      top: y + config.staffHeight / 2 - 18,
      left: 10,
    }}
  >
    <span className="sandbox-character-name">
      {character?.name || `干员${lineIndex + 1}`}
    </span>
    {character?.avatarUrl && (
      <img
        className="sandbox-avatar"
        src={character.avatarUrl}
        alt={`${character.name} 头像`}
        style={{ backgroundColor: getElementBackgroundColor(character.element) }}
        onError={(e) => {
          (e.target as HTMLImageElement).style.display = 'none';
        }}
      />
    )}
  </div>
);
```

**位置计算说明**：
- `y + config.staffHeight / 2` 是谱线中心 Y
- `- 18` 偏移是为了将容器（高度约 36px）对齐到谱线中心偏上

### 步骤 3：验证
