# 开发修改 0.2.7 - 画布技能按钮底色修正 + 图标存在时隐藏文字

## 约束

- 本计划仅修改画布技能按钮（`SkillButtonComponent`）的渲染逻辑
- 不修改拖拽算法、节点吸附逻辑、战斗计算逻辑
- 不修改头像路径规则
- 资源文件名严格遵循已有规范，**禁止臆造**

---

## 问题分析

### 问题一：画布技能按钮底色仍是 SKILL_COLORS

**当前代码（错误）**

`SkillButton.tsx` 第 27 行：

```tsx
style={{ backgroundColor: SKILL_COLORS[skillType] }}
```

这导致画布上所有相同技能类型的按钮颜色相同（如所有 B 技能都是蓝色），而应该按**干员的 element 属性**显示不同底色。

**根因**：`SkillButton` 接口没有存储 `element` 字段，导致组件无法获取干员属性。

### 问题二：图标存在时文字未隐藏

**当前代码（错误）**

```tsx
{skillIconUrl ? <img ... /> : null}
<span className="skill-label">{skillType}</span>
<span className="skill-character-name">{characterName}</span>
```

当 `skillIconUrl` 有值且图片加载成功后，`skill-label` 和 `skill-character-name` 仍然显示在图标上方，造成冗余文字。

---

## 技术方案

### 第一步：扩展 SkillButton 类型，增加 element 字段

修改 `src/types/index.ts`，在 `SkillButton` 接口增加可选字段：

```typescript
export interface SkillButton {
  // ... 既有字段不变 ...

  // 【新增】干员元素属性（用于渲染底色）
  element?: string;
}
```

### 第二步：在 useCanvasDrag 中注入 element

修改 `src/components/CanvasBoard/hooks/useCanvasDrag.ts`：

从 `characters` 数组中通过 `characterId` 查找干员的 `element`，注入到新建的 `SkillButton` 中：

```typescript
// 根据 characterId 查找干员的 element 属性
const characterElement = selectedCharacters.find(c => c.id === draggingState.characterId)?.element;

const newButton: SkillButton = {
  // ... 既有字段 ...
  skillIconUrl: resolveSkillIconUrl(draggingState.characterName, draggingState.skillType),
  element: characterElement,  // 【新增】
};
```

### 第三步：SkillButtonComponent 使用 element 底色

修改 `src/components/CanvasBoard/SkillButton.tsx`：

```tsx
import { getElementBackgroundColor } from '../../utils/assetResolver';

// backgroundColor 改为按 element 属性
style={{ backgroundColor: getElementBackgroundColor(button.element ?? '') }}
```

### 第四步：图标加载成功时隐藏文字

修改 `src/components/CanvasBoard/SkillButton.tsx`：

```tsx
// skillIconUrl 存在时：优先渲染图标，隐藏文字
{skillIconUrl ? (
  <img
    className="skill-icon"
    src={skillIconUrl}
    alt={SKILL_LABELS[skillType]}
    onError={(e) => {
      (e.target as HTMLImageElement).style.display = 'none';
      // 图片加载失败时：隐藏图标，显示文字（通过移除 hide-text 类）
    }}
    onLoad={(e) => {
      // 图片加载成功时：隐藏文字 span
      const parent = (e.target as HTMLImageElement).parentElement;
      const label = parent?.querySelector('.skill-label') as HTMLElement | null;
      const name = parent?.querySelector('.skill-character-name') as HTMLElement | null;
      label && (label.style.display = 'none');
      name && (name.style.display = 'none');
    }}
  />
) : null}
```

更简洁的做法：通过 CSS 配合 `.has-icon` 类来控制，在 JS 中只需给父容器添加/移除类名即可：

```tsx
const handleIconLoad = (e: React.SyntheticEvent<HTMLImageElement>) => {
  // 图片加载成功后隐藏文字
  const parent = (e.target as HTMLImageElement).parentElement;
  parent?.querySelectorAll('.skill-label, .skill-character-name')
    .forEach(el => (el as HTMLElement).style.display = 'none');
};

<img className="skill-icon" src={skillIconUrl} alt={...} onLoad={handleIconLoad} onError={...} />
```

### 第五步：CSS 调整

修改 `src/components/CanvasBoard/SkillButton.css`，确保图标层级最高，文字不受影响：

```css
.canvas-skill-button .skill-icon {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  border-radius: 50%;
  pointer-events: none;
  z-index: 2;  /* 图标在文字上方 */
}

.canvas-skill-button .skill-label,
.canvas-skill-button .skill-character-name {
  position: relative;
  z-index: 1;  /* 文字在图标下方，隐藏时不占位 */
}
```

## 防幻觉约束（显式声明）

| 约束项 | 说明 |
|---|---|
| 资源路径 | 严格使用 `resolveSkillIconUrl` 拼接，不自行发明 |
| 修改范围 | 仅涉及 `types/index.ts`、`useCanvasDrag.ts`、`SkillButton.tsx`、`SkillButton.css` |
| 回退策略 | 图标加载失败必须回退到显示文字，禁止文字被永久隐藏 |
| 字段可选 | `element` 为 `?`，不回退到修改既有 JSON 结构 |
| 按钮形状 | 画布技能按钮保持 `border-radius: 50%` 圆形不变 |
