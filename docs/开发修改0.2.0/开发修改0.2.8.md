# 开发修改 0.2.8 - 修复连续吸附时 y 轴偏移 Bug

## 约束

- 本计划仅修复 `snapToNearestNode` 的 y 轴重复偏移问题
- 不修改拖拽算法核心逻辑、不修改 `findNearestLine`
- 资源路径严格遵循规范，**禁止臆造**

---

## 问题分析

### Bug 表现

连续拖拽吸附多个按钮时，y 轴会在正确值（62）和错误偏移值（362）之间跳变：

```
吸附位置: {x: 234, y: 62}
吸附位置: {x: 256, y: 362}   ← 突然偏移了 300
吸附位置: {x: 278, y: 362}   ← 持续偏移
```

### 根因分析

**双重 groupOffset 计算导致 y 轴偏移**，分布在两处调用链中：

**第一处：`findNearestLine`（layout.ts 第 81 行）**

```typescript
const lineY = calculateLineY(config, lineIdx) + config.staffHeight / 2 + groupOffset;
//                                    ↑ staffHeight/2 + groupOffset 已加入
```

`findNearestLine` 返回的 `lineY` **已经包含** `groupOffset`（staff 内节点 y 偏移量）。

**第二处：`snapToNearestNode`（layout.ts 第 119-120 行）**

```typescript
const lineY = calculateLineY(config, lineIndex) + config.staffHeight / 2;
const groupOffset = getGroupOffset(config, staffIndex);
```

`snapToNearestNode` 内部**再次**计算 `lineY`（不含 groupOffset），然后在返回值中：

```typescript
return {
  snappedPosition: { x: snappedX, y: lineY + groupOffset },  // ← groupOffset 被加了两次！
  nodeIndex: targetNodeIndex,
};
```

**调用处（useCanvasDrag.ts 第 151-152 行）**

```typescript
const { snappedPosition } = snapToNearestNode(
  { x: mouseX, y: lineY },  // ← lineY 已经是含 groupOffset 的值
  ...
);
```

`findNearestLine` 返回的 `lineY` 已经带 groupOffset，但 `snapToNearestNode` **又加了一次** groupOffset 到 y 上，导致 y = 正确值 + groupOffset（约 +300），即为错误偏移值（62 + 300 = 362）。

**为何是间歇性出现**：当 `staffIndex = 0` 时，`getGroupOffset(config, 0) = 0`，此时无偏移，所以 y 值正确；当 `staffIndex > 0` 时，偏移显现。

### 结论

`snapToNearestNode` 对 y 轴重复应用了 `groupOffset`，应直接使用传入的 y 值（或传入的 y 已是含偏移的精确值）。

---

## 技术方案

### 修复：移除 snapToNearestNode 中重复的 groupOffset

**修改 `snapToNearestNode` 的返回值**（`src/utils/layout.ts` 第 145-148 行）：

将：
```typescript
return {
  snappedPosition: { x: snappedX, y: lineY + groupOffset },
  nodeIndex: targetNodeIndex,
};
```

改为：
```typescript
return {
  // x 需要节点间距计算，y 直接使用传入值（findNearestLine 返回的 lineY 已含 groupOffset）
  snappedPosition: { x: snappedX, y: position.y },
  nodeIndex: targetNodeIndex,
};
```

**为什么 x 可以用计算值**：x 是鼠标在画布内的相对位置，每次根据 `mouseX` 重新计算节点索引，是正确的。  
**为什么 y 直接用 `position.y`**：传入的 `position.y` 就是 `findNearestLine` 返回的 `lineY`（含 groupOffset），直接使用即可，不需要再偏移。
