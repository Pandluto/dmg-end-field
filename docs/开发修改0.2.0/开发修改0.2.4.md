# 开发修改 0.2.4 - 技能沙盒头像与四技能图标渲染

## 约束

- 本计划**仅针对技能沙盒（SkillSandbox）**的头像与技能图标渲染改造
- **选人面板（SelectionPanel）不在本次修改范围内**，保持原样
- 不修改拖拽算法、节点吸附逻辑、战斗计算逻辑
- 不修改 `snapToNearestNode`、`findNearestLine`、`useCanvasDrag` 等核心逻辑
- 所有新增图片路径必须基于 `public/assets/avatars/` 实际目录结构，**禁止臆造不存在资源**
- 每个 `<img>` 节点必须具备 `onError` 回退策略，防止资源缺失时 UI 断裂
- 新增字段均为可选，保持向后兼容，不破坏现有 JSON 数据结构
---

## 问题分析

### 当前状态

**技能沙盒（SkillSandbox）**
- 当前渲染角色名（`.sandbox-character-name`）+ 四技能按钮（仅文字 A/B/E/Q）
- 头像区域**不存在**，技能图标**不存在**
- 渲染入口：`src/components/CanvasBoard/SkillSandbox.tsx` 第 18-58 行
- 当前技能按钮样式：`border-radius: 50%`（圆形） + `border` + `SKILL_COLORS` 背景

**已知 element 枚举（来源 `types/index.ts`）**

| element 值 | 属性描述 | 头像背景色（半透明） |
|---|---|---|
| `physical` | 物理 | 淡灰色 `rgba(210, 210, 210, 0.75)` |
| `ice` | 寒冷 | 冰蓝色 `rgba(180, 220, 255, 0.75)` |
| `fire` | 炎热 | 火红色调 `rgba(255, 180, 140, 0.75)` |
| `electric` | 电气 | 电气黄 `rgba(230, 230, 120, 0.75)` |
| `ether` | 异核 | 异核紫 `rgba(200, 160, 255, 0.75)` |
| `nature` | 自然 | 自然绿 `rgba(120, 200, 140, 0.75)` |

**管理员**：`element: "physical"` → 淡灰色背景
**别礼**：`element: "ice"` → 冰蓝色背景
**汤汤**：`element: "ice"` → 冰蓝色背景

### 根因

渲染层缺少头像 `<img>` 与技能图标 `<img>` 节点；数据层缺少派生字段（`avatarUrl`、`skillIconMap`）。

## 技术方案

### 第一步：建立资源路径解析工具

新增 `src/utils/assetResolver.ts`，集中管理路径拼接规则：

```typescript
// src/utils/assetResolver.ts

import type { SkillType } from '../types';

/**
 * 角色头像资源路径解析
 * 规范路径: /assets/avatars/<characterName>/<characterName>.png
 * @param characterName - 角色名称（URL 编码）
 */
export function resolveAvatarUrl(characterName: string): string {
  return `/assets/avatars/${encodeURIComponent(characterName)}/${encodeURIComponent(characterName)}.png`;
}

/**
 * 技能图标资源路径解析
 * 规范路径: /assets/avatars/<characterName>/skills/<skillType>.png
 * 技能类型 A/B/E/Q 对应 普攻战技/连携技/终结技/爆发技
 * @param characterName - 角色名称
 * @param skillType     - A | B | E | Q
 */
export function resolveSkillIconUrl(characterName: string, skillType: SkillType): string {
  return `/assets/avatars/${encodeURIComponent(characterName)}/skills/${skillType}.png`;
}

/**
 * 基于 element 属性返回半透明背景色
 * 用于头像区域底色，适配半透明 PNG 贴图
 */
export function getElementBackgroundColor(element: string): string {
  const colorMap: Record<string, string> = {
    physical:  'rgba(210, 210, 210, 0.75)',   // 淡灰色
    ice:       'rgba(180, 220, 255, 0.75)',   // 冰蓝色
    fire:      'rgba(255, 180, 140, 0.75)',   // 火红色调
    electric:  'rgba(230, 230, 120, 0.75)',   // 电气黄
    ether:     'rgba(200, 160, 255, 0.75)',   // 异核紫
    nature:    'rgba(120, 200, 140, 0.75)',   // 自然绿
  };
  return colorMap[element] ?? 'rgba(200, 200, 200, 0.75)';
}
```

### 第二步：扩展 Character 类型定义

在 `src/types/index.ts` 的 `Character` 接口增加可选派生字段：

```typescript
// src/types/index.ts

// 既有字段不变，仅新增派生字段
interface Character {
  // ... 既有字段 ...

  // 【新增】派生字段（由 AppContext 加载后注入，非 JSON 直写）
  avatarUrl?: string;
  skillIconMap?: Partial<Record<SkillType, string>>;
}
```

### 第三步：在 AppContext 中统一注入派生字段

修改 `src/context/AppContext.tsx` 第 131-139 行（角色 JSON 加载后）：

```typescript
// 统一注入派生路径，避免多组件重复拼接
const enrichedCharacters: Character[] = characters.map((char) => ({
  ...char,
  avatarUrl: resolveAvatarUrl(char.name),
  skillIconMap: {
    A: resolveSkillIconUrl(char.name, 'A'),
```

### 第四步：改造 SkillSandbox 渲染

修改 `src/components/CanvasBoard/SkillSandbox.tsx`：

**头像渲染**：插入到 `.sandbox-character-header` 中的 `.sandbox-character-name` **右侧**（使用 flex 布局）

**四技能图标**：在圆形按钮内部优先渲染 `<img>`，**图片缺失时退回 A/B/E/Q 文字**（不退背景色）

**底色**：头像容器背景使用 `getElementBackgroundColor(character.element)`

关键代码结构示例：

```tsx
// .sandbox-character-header 使用 flex 纵向排列：name 在左，avatar 在右
<div className="sandbox-character-header">
  <span className="sandbox-character-name">{character.name}</span>
  {character.avatarUrl && (
    <img
      className="sandbox-avatar"
      src={character.avatarUrl}
      alt={`${character.name} 头像`}
      // 加载失败时隐藏，避免显示断裂图
      onError={(e) => { (e.target as HTMLImageElement).style.display = 'none'; }}
    />
  )}
</div>

// 四技能按钮内部：优先图标，失败退文字
<div
  className={`sandbox-skill-button skill-${skillType.toLowerCase()}`}
  style={{ backgroundColor: SKILL_COLORS[skillType] }}
  onMouseDown={(e) => { onDragStart(character.id, character.name, skillType, index, e); }}
>
  {character.skillIconMap?.[skillType] ? (
    <img
      className="skill-icon"
      src={character.skillIconMap[skillType]}
      alt={SKILL_LABELS[skillType]}
      // 图片缺失时退回文字：onError 将 img 隐藏，文字 span 仍显示
      onError={(e) => { (e.target as HTMLImageElement).style.display = 'none'; }}
    />
  ) : null}
  {/* 无论图片加载成功与否，文字始终显示（作为兜底） */}
  <span className="skill-label">{skillType}</span>
</div>
```

**布局调整要点**：
- `.sandbox-character-header` 改为 `display: flex; align-items: center; gap: 8px;`
- `.sandbox-avatar` 尺寸建议 `36px × 36px`，`border-radius: 50%`，`object-fit: cover`
- `.sandbox-character` 内部通过 padding/gap 调整比例，避免头像与技能按钮拥挤
- 技能按钮**保持 `border-radius: 50%`（圆形）**和 `border` 不变

## 防幻觉约束（显式声明）

| 约束项 | 说明 |
|---|---|
| 资源路径 | 严格遵循 `/assets/avatars/<角色名>/<角色名>.png` 规范，技能图标为 `skills/<skillType>.png`，不自行发明文件名 |

| 修改范围 | **仅限 SkillSandbox**（选人面板不在本次范围），仅限渲染层与数据派生层，不碰核心算法 |

| 字段可选 | 新增字段均为 `?`，不强制 JSON 结构变更 |
| 头像位置 | 头像必须位于 `.sandbox-character-name` 的**右侧**，通过 flex 布局实现 |
