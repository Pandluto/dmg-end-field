# 草稿工作表拆分手工冒烟

本清单用于后续人工验证 2026-07-21 的 Weapon、Buff、Equipment、Operator Studio 页面拆分，以及旧 Damage Sheet 下线后的入口状态。自动验证已经覆盖类型检查、现有业务测试和 Web 生产构建；本清单只检查真实界面交互。

## 运行约束

- 优先复用当前主进程；不要重启或占用它的端口。
- 若只测 UI，可在独立临时分支使用未被占用的端口启动 Vite，不启动第二个 Electron 主进程。
- 测试数据使用临时草稿，避免覆盖正式本地库。

## Weapon Sheet

1. 打开武器工作表，确认草稿、技能、等级行正常展示。
2. 选择单元格，确认地址栏、公式编辑、行内编辑仍同步。
3. 折叠/展开技能与等级，拖动可排序节点。
4. 新建、保存、覆盖保护、规范化、分享导入导出各执行一次。
5. 打开 Buff 编辑抽屉并保存一次修改。

## Buff Sheet

1. 打开 Buff 工作表，确认分组头、项目合并行、效果明细列宽与对齐正常。
2. 点击效果单元格，确认地址、公式栏和选中态一致。
3. 折叠/展开项目，拖动草稿、项目和效果节点。
4. 新建、保存、覆盖确认、规范化、撤回和分享导入导出各执行一次。
5. 打开效果抽屉并保存一次 modifier 和 extra hit。

## Equipment Sheet

1. 打开装备工作表，确认套装、装备、固定属性、效果和等级行正常展示。
2. 编辑数值与公式栏，确认选中单元格和预览图更新。
3. 折叠/展开套装、装备和效果；使用右键菜单执行一个无破坏操作。
4. 新建、规范化、保存确认和分享导入导出各执行一次。
5. 打开三件套 Buff 抽屉并保存一次修改。

## Operator Studio

1. 从“编辑干员”进入 Operator Studio，确认基础信息、属性、技能、Hit 与 Buff 分区正常展示。
2. 新建草稿，编辑干员名称、技能与 Hit，确认技能筛选、复制、拖动排序及“整理编号”行为正常。
3. 分别验证首次保存、同 ID 覆盖保护、确认覆盖、另存为以及 `Ctrl+S`。
4. 从本地库导入一个草稿，再打开本地库管理并删除一个临时草稿。
5. 分别导出当前干员与整个干员库，复制分享 JSON，并重新导入分享文件。
6. 检查头像和技能图标路径搜索、Markdown 预览与 Buff 编辑抽屉。

## 画布入口

1. 确认底部工具栏不再显示“表格”按钮。
2. 确认“计算伤害”仍能打开独立伤害报表/PPT 页面。

## 批量 Buff 编辑工作台

1. 从工作台切换到批量 Buff 编辑，确认谱线、技能按钮位置、图标和右侧面板正常显示。
2. 使用单击与框选选择技能按钮，并用干员快捷按钮批量选择、取消选择。
3. 分别进入筛选、增加、删减和编辑目录模式；验证确认与 `Esc` 取消不会残留草稿标记。
4. 在增加模式按 `Tab` 打开候选面板，切换 Buff 组、武器、装备和异常状态入口并添加候选 Buff。
5. 给多个技能按钮批量增加 Buff，再批量删减及编辑共同/剩余 Buff，确认计数和按钮标记同步更新。
6. 缩放窗口宽度，确认右侧快捷按钮和网格位置仍随布局更新。

## 通过标准

- 四个编辑页及批量 Buff 工作台无白屏、无新增控制台错误。
- 单元格地址、选中态、折叠、拖拽、公式编辑与保存行为和拆分前一致。
- 旧 Damage Sheet 入口已移除，伤害报表/PPT 能力仍保留。
