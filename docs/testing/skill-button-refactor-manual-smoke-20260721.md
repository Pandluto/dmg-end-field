# SkillButton 核心文件拆分人工冒烟手册

## 目的

本手册用于人工确认 `SkillButton` 从巨型单文件拆分后，按钮交互、Buff 管理、伤害详情和持久化行为没有发生回归。

对应分支：`codex/core-file-slimming-research-20260721`

主要提交：

- `c202338 refactor(skill-button): extract damage modal view`
- `bee921d refactor(skill-button): split runtime responsibilities`

本轮拆分后的主要文件：

- `src/components/CanvasBoard/SkillButton.tsx`：组件组装与 Workbench 展示。
- `src/components/CanvasBoard/useSkillButtonRuntime.ts`：Buff、搜索、面板数据及持久化状态。
- `src/components/CanvasBoard/useSkillButtonDamageRuntime.ts`：伤害计算、Hit 和异常伤害派生状态。
- `src/components/CanvasBoard/useSkillButtonInteractions.ts`：单击、双击、长按和图标状态。
- `src/components/CanvasBoard/SkillButtonDamageModal.tsx`：伤害弹窗视图。
- `src/components/CanvasBoard/SkillButtonInfoModal.tsx`：技能信息、抗性和已选 Buff 视图。
- `src/components/CanvasBoard/SkillButtonContextMenu.tsx`：右键菜单。

## 启动注意事项

完整的排轴和 SkillButton 详情依赖 Electron 提供的 SQLite local-data transport。只运行 Vite UI 时可以检查页面加载和普通编辑页，但点击“开始排轴”会提示：

> 创建 SQLite 工作区失败：Workbench local-data transport is unavailable to this caller.

这是纯 UI 模式的预期限制，不是本次拆分造成的错误。

进行完整人工测试时：

1. 使用本手册对应的独立 worktree 和分支。
2. 确保只启动一个 Electron 开发实例，避免和已有的 3030 主进程冲突。
3. 如果主开发实例仍在运行，不要直接在另一个 worktree 执行 `npm run electron:dev`。
4. 选择方便的时间关闭原开发实例后，再在测试 worktree 中执行：

   ```powershell
   npm run electron:dev
   ```

5. 测试完成后关闭该测试实例，再恢复平时使用的开发实例。

如果只想检查 Web 页面能否加载，可使用单独端口：

```powershell
npx vite --host 127.0.0.1 --port 3031 --strictPort
```

此方式不能完成下面的排轴与 SkillButton 详情测试。

## 测试数据准备

建议准备一条最小但覆盖完整的排轴：

- 至少一名干员。
- 至少一个普通技能按钮，并且技能包含一个以上 Hit。
- 至少一个普通被动 Buff。
- 至少一个可计层 Buff，最大层数大于 1。
- 至少一个异常伤害或异常状态快照。
- 技能按钮具有可用的面板快照和伤害模板。

测试前记录该技能原本的总伤、各 Hit 伤害和 Buff 层数，方便对照。

## 一、按钮本体与基础交互

- [ ] 排轴中的技能按钮位置、尺寸、图标、技能名称和元素颜色正常。
- [ ] 图标加载失败时能显示技能字母兜底，不出现空白圆点。
- [ ] 单击按钮不会意外打开详情或触发拖动。
- [ ] 双击按钮能够打开对应的技能详情页。
- [ ] 长按约 200ms 后能够拖动按钮。
- [ ] 禁止拖动状态下，长按不会触发拖动。
- [ ] 浏览模式下不会触发编辑、右键菜单或拖动行为。
- [ ] Inspect 模式能显示期望伤害和非暴击伤害摘要。

## 二、右键菜单

- [ ] 右键技能按钮后菜单位置正确，没有被面板遮挡。
- [ ] “取消”可以关闭菜单。
- [ ] “编辑”子菜单能列出可替换技能。
- [ ] 替换技能后按钮类型、名称和运行时技能 ID 正确更新。
- [ ] “复制”能复制按钮且不会修改原按钮。
- [ ] “删除”能按原有规则删除按钮。
- [ ] 锁定按钮后，删除限制与拆分前一致。

## 三、技能信息面板

- [ ] 角色名、技能类型、技能名、技能等级模式显示正确。
- [ ] 干员索引、节点索引和节点编号显示正确。
- [ ] 锁定按钮可以在锁定与解锁间切换，图标和文案同步变化。
- [ ] “目标抗性”可以展开和收起。
- [ ] 修改物理、灼热、电磁、寒冷、自然抗性后，伤害结果立即更新。
- [ ] 关闭详情再重新打开，目标抗性仍保持刚才的值。
- [ ] 已选 Buff 列表与按钮实际绑定的 Buff 一致。
- [ ] 可计层 Buff 的减层、加层、最小值和最大值限制正常。
- [ ] 右键 Buff 能移除该 Buff。
- [ ] 已选状态、异常状态快照和异常伤害显示正确。
- [ ] 右键状态或异常卡片可以移除对应项目。
- [ ] “关闭”能够返回主界面，且不丢失已保存数据。

## 四、Buff 搜索与资源栏

- [ ] 技能详情打开时，按 `Tab` 能打开 Buff 搜索层。
- [ ] 搜索层打开后，按 `Tab` 能依次切换 Buff组、干员、武器、装备、异常伤害、异常状态区和状态区。
- [ ] 按 `Esc` 先关闭搜索层；搜索层关闭后再次按 `Esc` 能关闭技能详情。
- [ ] Buff组、干员、武器、装备关键词搜索结果正常。
- [ ] 干员头像筛选、天赋/潜能/技能筛选正常。
- [ ] 潜能 Buff 的显示条件与当前干员潜能一致。
- [ ] 点击搜索结果后 Buff 能加入当前技能按钮。
- [ ] “附近 Buff”只显示相邻节点可用且尚未选择的 Buff。
- [ ] 点击附近 Buff 后能加入已选 Buff。
- [ ] 右侧“已选 Buff”资源栏内容与技能信息面板一致。
- [ ] 异常伤害、异常状态快照和状态资源栏的添加、引用与删除限制正常。

## 五、伤害面板与 Hit 调整

- [ ] 总伤期望、总伤暴击、总伤非暴击与拆分前记录一致。
- [ ] 总伤非暴击步骤中的各 Hit 和异常伤害段没有遗漏或重复。
- [ ] 普通 Hit 列表名称、Buff 数量、倍率和三种伤害值正确。
- [ ] 异常伤害段名称、类型、倍率和三种伤害值正确。
- [ ] 点击普通 Hit 后能显示对应详情，不会错误选中异常伤害段。
- [ ] 点击异常伤害段后能显示对应详情，不会错误选中普通 Hit。
- [ ] 禁用普通 Hit 后，该段样式变为禁用且总伤扣除该段。
- [ ] 重新启用普通 Hit 后，总伤恢复。
- [ ] 禁用和恢复异常伤害段时，总伤变化正确。
- [ ] Hit 详情中的 Buff 微调按钮可以临时启停对应 Buff。
- [ ] 可计层 Buff 在单个 Hit 中加层、减层后，只影响目标计算段。
- [ ] “重置微调”能恢复该计算段的 Buff 状态和层数。
- [ ] “全部启用”“全部停用”“重置层数”“清空 Buff”行为与拆分前一致。

## 六、公式展开区

- [ ] 普通 Hit 的“展开计算过程”与“收起计算过程”正常。
- [ ] 面板属性、有效 Buff、倍率区、加成区、增幅区、易伤区、脆弱区、连击区、失衡区、防御区和抗性区均有内容。
- [ ] 普通 Hit 展开的最终结果与卡片伤害一致。
- [ ] 异常伤害段的“展开异常计算过程”与“收起异常计算过程”正常。
- [ ] 异常伤害的源石技艺强度、等级系数和异常专属信息显示正确。
- [ ] 额外伤害 Buff 的来源 Buff、失衡值和冷却文案显示正确。
- [ ] 在普通 Hit 与异常伤害段之间切换时，展开状态不会串到错误的计算段。

## 七、持久化与重新进入

完成若干修改后关闭技能详情、切换页面或重启测试实例，再检查：

- [ ] 目标抗性保留。
- [ ] 全局停用的 Buff 保留。
- [ ] 各计算段手动停用的 Buff 保留。
- [ ] 各计算段单独调整的 Buff 层数保留。
- [ ] 手动禁用的 Hit 或异常伤害段保留。
- [ ] 已选异常状态和异常伤害保留。
- [ ] 总伤与关闭前一致。

## 八、错误检查

- [ ] 操作过程中没有白屏或 React 错误边界。
- [ ] 控制台没有新增 `TypeError`、Hooks 顺序错误或状态更新死循环。
- [ ] 快速切换 Hit、连续开关 Buff、连续打开关闭详情时没有明显卡死。
- [ ] 关闭详情后没有残留遮罩、浮层或无法点击的透明区域。
