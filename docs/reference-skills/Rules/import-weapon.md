# 武器导入 SOP

本文档是武器导入任务的标准 SOP。

## 任务目标

将用户提供的武器资料整理为统一结构，并落盘到：
- `data/weapons/{武器名}.json`
- `data/weapons/{武器名}.md`

其中：
- `json` 是主源
- `md` 必须由 `json` 派生

## 输入要求

可接受输入包括：
- 抓包返回的原始 JSON
- 接口响应片段
- 表格化整理结果
- 用户手动贴出的结构化或半结构化文本
- 可直接确认的武器基础规则

理想情况下应尽量覆盖：
- 基础信息：名称、英文名、稀有度、类型、描述、备注
- 攻击成长：1-90 级每级攻击力
- 阶段信息：0-4 阶段及等级需求
- 技能信息：3 个武器技能的阶段范围与 1-9 级效果
- 珠子信息：起始编码、满级编码、说明

## 输出路径

统一输出到：
- `data/weapons/{武器名}.json`
- `data/weapons/{武器名}.md`

不要再写到旧路径：
- `weapons/`
- 绝对路径目录说明

## 执行步骤

1. 读取用户提供的武器资料
2. 判断是否足以构成单把武器的标准数据
3. 先按字段语义整理原始资料，再映射到标准结构
4. 先生成 `data/weapons/{武器名}.json`
5. 再根据 `json` 生成 `data/weapons/{武器名}.md`
6. 校对字段名、数值格式、路径口径

## 基本规则

- 优先保留原始资料语义，不擅自改写效果
- 百分比一律转小数
- 等级、阶段、技能等级字段统一使用字符串 key
- 不把原始抓包字段直接当最终字段名
- 不臆造未提供的技能等级效果
- 可由基础规则直接补齐的信息允许补齐
- 术语和数值口径优先服从 `Rules/基础知识.md`

## 数值规范

- `5%` -> `0.05`
- `36.0%` -> `0.36`
- `100.8%` -> `1.008`
- 攻击成长使用整数
- 技能等级固定为 `1` 到 `9`
- 阶段固定为 `0` 到 `4`

## 字段提取说明

### 基础信息
- `name`：武器中文名
- `nameEn`：英文名；抓包没有可留空字符串或按你的习惯处理
- `rarity`：稀有度数字
- `type`：武器类型
- `description`：武器描述
- `remark`：备注，可选

### attackGrowth
- 原始资料中的 1-90 级攻击力整理为：

```json
{
  "1": 51,
  "2": 56,
  "...": 0,
  "90": 500
}
```

- key 必须是字符串
- value 必须是数值
- 不要只保留几个关键等级，要求尽量保留完整成长表

### phases
- `requirement`: 该阶段的等级需求
- 阶段固定为 `0` 到 `4`
- 只写升级门槛，不在这里重复写技能等级范围

### matrix（珠子）
- `name`: 珠子（俗称）
- `start`: 起始编码，如 `111`
- `max`: 满级编码，如 `663`
- `note`: 编码规则说明

珠子信息默认按通用规则处理：
- `matrix` 建议始终保留，不要因为抓包里没写就省略
- `name` 一般直接写 `珠子`
- `start` 可按通用起始值写 `111`
- `max` 可按通用满级值写 `663`
- `note` 可写为 `珠子编码如663表示三个技能等级分别为6、6、3`

也就是说，`matrix` 属于可由基础规则直接补齐的信息，不要求必须来自抓包原文。

### skills
固定包含 3 个技能槽位：
- `skill1`
- `skill2`
- `skill3`

3 个槽位都允许自由化，不再要求：
- 第一技能必须是某个固定属性技能
- 第二技能必须是攻击提升
- 第三技能必须是某个固定专武效果

每个技能包含：
- `name`: 技能名称
- `statType`: 技能类型或主效果类型，例如 `strength`、`agility`、`atkPercent`、`memoryStrength`、`special`
- `phases`: 各阶段的基础等级和上限等级
- `levels`: 1-9级的效果数据

#### skill.phases
- `baseLevel`: 武器提供的基础等级
- `maxLevel`: 该阶段的等级上限

#### skill.levels
- `value`: 单一数值型效果时使用
- `passive`: 无条件生效的效果（面板直接加成，参与计算）
- `description`: 效果描述（条件生效部分用文字记录，不参与面板计算）

#### 无条件生效 vs 条件生效（重要）

第三技能通常包含两种效果类型，必须区分：

**无条件生效（passive）**：
- 直接加到角色面板
- 无需任何触发条件
- 参与面板计算
- 例如：攻击力+7%、源石技艺强度+30

**条件生效**：
- 需要特定条件触发
- 触发后一定时间内生效
- **只用文字记录在description中，不参与面板计算**
- 例如：施加破防时终结技伤害+33.6%

## 原始字段到标准字段的整理方向

如果原始字段名不是最终字段名，按语义整理：

| 原始资料语义 | 标准字段 |
|------|------|
| 武器中文名 | `name` |
| 武器英文名 | `nameEn` |
| 星级/稀有度 | `rarity` |
| 武器类型 | `type` |
| 描述文案 | `description` |
| 备注/专武说明 | `remark` |
| 1-90级攻击 | `attackGrowth` |
| 升阶段等级要求 | `phases` |
| 珠子起始/满级/说明（可按基础规则补齐） | `matrix` |
| 第一个武器技能 | `skills.skill1` |
| 第二个武器技能 | `skills.skill2` |
| 第三个武器技能 | `skills.skill3` |

如果原始字段命名很乱，先按意义整理，再落标准结构，不要机械照抄原字段名。

## 武器技能等级说明

```
武器技能等级 = 武器技能基础等级 + 珠子技能等级
```

但最终等级不能超过当前阶段的上限等级。

珠子编码示例：
- `111` = 第一个技能1级 / 第二个技能1级 / 第三个技能1级
- `663` = 第一个技能6级 / 第二个技能6级 / 第三个技能3级
- 无珠子 = 三个技能等级均为0

## 输出 md 的写法要求

`md` 文件用于人读，内容应从 `json` 派生，建议包含：
- 标题和一句描述
- 基础信息表
- 攻击力成长表
- 升阶段表（必须包含技能基础等级/上限等级）
- 珠子说明（必须包含编码规则和计算公式）
- 三个武器技能分节展示

## 基质（珠子）

基质（俗称珠子）是武器的附加系统，提供额外的技能等级加成。

**珠子编码规则：**
- 三位数字分别对应三个武器的技能等级
- 例如：663 表示 技能1名 6级 / 技能2名 6级 / 技能3名 3级
- 珠子等级上限：663（三个技能全满）
- 珠子起始等级：111（三个技能全为1级）
- 可无基质

**武器技能最终等级计算：**
```
武器技能等级 = 武器技能基础等级 + 珠子技能等级
```
但最终等级不能超过当前阶段的上限等级。

## 缺失信息处理

- 缺少 `remark`：可省略
- 缺少 `nameEn`：可留空或按你的习惯处理
- 缺少 `type`：可留空字符串，等待后续补录
- 缺少 `matrix` 原始来源：直接按通用规则补齐，不视为缺失
- 缺少部分技能描述：保留已有数值，`description` 可简写或省略
- 缺少部分攻击成长：优先提示缺失；如果用户明确允许，再按已有来源补录
- 如果 3 个技能数据不完整，不要假设未给出的等级效果

## 禁止事项

- 不要把抓包原始字段原封不动塞进最终 JSON
- 不要混入与标准结构无关的调试字段
- 不要把百分比直接写成 `36`，应写成 `0.36`
- 不要把 `md` 写成比 `json` 更多的事实来源
- 不要在未确认的情况下补出不存在的技能等级数据
- 不要把第二技能、第三技能写死为固定模板
- 不要因为原始资料缺少珠子字段就把 `matrix` 整段省略
- **不要在升阶段表格中缺少"武器技能"列**
- **不要在基质部分缺少编码规则和计算公式**
- **不要用简化的表格格式，必须对齐宏愿.md的完整格式**
- **不要混淆第三技能的无条件生效和条件生效效果**
- **不要把无条件效果和条件效果分开写，必须合并在同一行**
- **不要使用 `**条件效果**：` 单独段落格式**
- **不要只写 Rank 1 和 Rank 9，必须写出完整的 1-9 级**

## 完成标准

完成一次武器导入，至少满足：
- `data/weapons/{武器名}.json` 已生成或更新
- `data/weapons/{武器名}.md` 已生成或更新
- JSON 结构符合本规范
- MD 内容与 JSON 一致
- 百分比、小数、阶段、等级格式统一
- 路径写法正确
- **MD 升阶段表格包含"武器技能"列**
- **MD 基质部分包含编码规则和计算公式**
- **MD 格式对齐宏愿.md的完整格式**
- **第三技能无条件效果和条件效果合并在同一行**
- **面板计算只考虑passive部分，不考虑条件生效**
