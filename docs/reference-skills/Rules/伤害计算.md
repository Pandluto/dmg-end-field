# 伤害计算（草稿）
> 本文档定义伤害计算的最小可用口径（V0 草稿），用于先落地单段/多段伤害结果。  
> 未在本文明确的机制，暂不纳入计算。

## 1. 规则来源与优先级
- 伤害主公式：`Rules/基础知识.md`
- 角色面板数据：`Rules/面板计算.md` 对应口径
- 技能倍率数据：`data/characters/{角色名}.json`
- 文本输出： 

冲突时优先级：
1. `Rules/基础知识.md`
2. `Rules/面板计算.md`
3. `Rules/伤害计算.md`（本文）

## 2. 默认规则
- 默认技能等级取 `9`
- 若输入明确说明“专3”，则取 `M3`
- 默认按当前面板（含装备、武器、珠子、潜能）计算
- 百分比统一按小数处理（例如 `53.8% = 0.538`）

## 3. 输入项（V0）
- `角色名`
- `技能类型`：普通攻击 / 战技 / 连携技 / 终结技
- `技能倍率等级`：默认 `9`，可显式指定 `M3`
- `面板ATK`
- `暴击率`、`暴击伤害`
- 伤害加成项：
  - 元素/类型伤害加成（如寒冷、电磁等）
  - 技能类型加成（`skillDmgBonus` / `chainSkillDmgBonus` / `ultimateDmgBonus`）
  - 所有伤害加成（`allSkillDmgBonus`）

## 4. 段伤公式
本节严格对齐 `Rules/基础知识.md` 的伤害公式，并采用如下常规伤害乘区顺序。  
记第 `i` 段倍率为 `m_i`：

```text
段伤_i = 基础伤害区 × 暴击区 × 伤害加成区 × 伤害减免区
       × 增幅区 × 脆弱区 × 易伤区
       × 防御区 × 失衡易伤区 × 抗性区
       × 连击增伤区 × 特殊乘区

基础伤害区 = ATK × m_i
```

## 4.1 分区维护规范（文档与代码统一）
为便于后续维护，每一个“区”必须独立成段，且代码实现保持一一对应：
- 文档：每个区有单独章节，写清定义、取值、默认值、数据来源
- 代码：每个区有单独计算块（单独变量），最后统一汇总到总公式
- 未实现区：明确写默认值（通常为 `1` 或 `0`），禁止隐式省略
- 每个区必须标记归属：角色侧 / 敌方侧
- 代码与文本先分别统计“角色侧区”和“敌方侧区”，再代入总公式

## 4.2 各区定义（每区单独大段）
### A. 基础伤害区
- 定义：`基础伤害区 = ATK × m_i`
- 数据来源：面板 `ATK` + 角色 `json` 段倍率
- 默认值：无（必算）

### B. 暴击区
- 非暴击：`1`
- 暴击：`1 + CritDmg`
- 期望：`1 + CritRate × CritDmg`
- 数据来源：面板暴击率 / 暴伤

### C. 伤害加成区
- 定义：`1 + 元素(或物理)伤害加成 + 技能类型伤害加成 + 所有伤害加成`
- 当前字段：`physical/fire/electric/ice/ether + skill/chain/ultimate + allSkillDmgBonus`
- 默认值：`1`
- 归属：角色侧
- 数据来源（按口径汇总）：
  - 角色潜能（是否命中基于角色当前潜能）
  - 角色天赋
  - 武器技能3(独立生效的效果单独呈现)
  - 装备效果
  - 来自其他角色的加成
- 同源同类规则：同一来源链路内，按“可命中等级”取最高级效果
- 技能类型命中规则：
  - 伤害类型为【普通攻击 / 战技 / 连携技 / 终结技】时，命中角色自身对应类型加成
  - 例如战技伤害只命中战技类型加成（再叠加元素/全伤等公共项）
- 加成分类：
  - 无条件加成：默认计入
  - 有条件加成：默认不触发，仅在用户显式选择触发时计入
- 命中判断口径（简要）：
  - 读取角色/武器的计算口径备注
  - 判断是否计入本次伤害时，不按“来源技能标签”直接决定
  - 必须满足：`该加成可作用于本次伤害（元素命中/类型命中）`
  - 即：`是否命中 = 加成可作用性`
  - 例如：`寒冷伤害加成` 作用于所有 `寒冷伤害`
  - 状态定义（必须严格区分）：
    - `命中`：该加成可作用于本次伤害（元素/类型匹配），表示“有资格”
    - `已触发`：该加成在当前场景触发条件成立，表示“当前生效”
  - 计入规则：
    - `是否计入 = 命中 × 已触发`
    - 仅“命中”不代表已生效，未触发时不得计入

### D. 伤害减免区
- 当前状态：未实现精细模型
- 默认值：`1`
- 归属：角色侧

### E. 增幅区
- 定义：`1 + Σ增幅效果`
- 当前字段：
  - `physicalAmplify`
  - `magicAmplify`
  - `fireAmplify`
  - `electricAmplify`
  - `iceAmplify`
  - `natureAmplify`
- 当前状态：已接入 Buff 驱动
- 当前代码口径：
  - `physical` 伤害只吃 `physicalAmplify`
  - 元素伤害吃 `对应元素Amplify + magicAmplify`
- 叠加规则：多个增幅 Buff 直接相加，最终按 `× (1 + amplifyRate)` 进入总公式
- 默认值：`1`
- 归属：角色侧

### F. 脆弱区
- 定义：`1 + Σ脆弱效果`
- 说明：脆弱为“目标受到特定属性伤害提高”
- 当前状态：未接入字段
- 默认值：`1`
- 归属：敌方侧

### G. 易伤区
- 当前状态：未实现
- 默认值：`1`
- 归属：敌方侧

### H. 防御区
- 当前口径：干员对敌人造成伤害时固定取 `0.5`
- 说明：后续可扩展精细防御模型
- 归属：敌方侧

### I. 失衡易伤区
- 当前状态：未实现
- 默认值：`1`
- 归属：敌方侧

### J. 抗性区
- 当前口径：默认 `1`
- 说明：后续可扩展敌人属性抗性
- 归属：敌方侧

### K. 连击增伤区
- 定义：`连击增伤区 = 1 + 连击增伤`
- 归属：角色侧
- 机制说明：
  - 连击是一种特殊增伤效果。
  - 施加连击效果后，全队的下个战技或终结技伤害提高。
  - 其中对战技的提高幅度更大，触发后效果清空。
  - 同类效果可以叠加，最多 `4` 层，但叠加为非线性衰减，不是简单线性相加。
- 当前已确认来源：
  - 黎风的连携技
  - 秋栗解锁天赋 `心流时间` 后的终结技
- 分层口径：
  - 战技伤害提升：`30% / 45% / 60% / 75%`
  - 终结技伤害提升：`20% / 30% / 40% / 50%`
- 命中规则：
  - 只作用于“下一个”战技或终结技。
  - 对普通攻击、连携技、异常伤害、额外 hit 是否命中，需要单独明确，不得默认继承。
- 默认值：
  - 无连击效果时取 `1`

### L. 特殊乘区
- 当前状态：未实现
- 默认值：`1`
- 归属：角色侧

## 5. 多段总伤
若技能为多段，逐段独立计算后求和：

```text
总伤(不暴击) = Σ 非暴击段伤_i
总伤(暴击)   = Σ 暴击段伤_i
总伤(期望)   = Σ 期望段伤_i
```

- 有条件项输出时必须同时标注“命中状态”和“触发状态”，禁止二者混写

## 7. 技能段位映射（V0）
- 若倍率键为 `hit1/hit2/hit3...`，按自然顺序输出为 `1hit/2hit/3hit...`
- 若倍率键为 `damage`（单段），输出为 `1hit`
- 若为特殊键（如 `slash1/slash2/stabTotal`），在实现中定义映射顺序后再统一输出 `1hit/2hit/...`

## 8. 示例（V0）
### 8.1 单段示例（默认9级）
```text
输入: ATK=3000, m_1=2.40, 伤害加成区=1.50, 暴击率=0.30, 暴伤=0.50
非暴击1hit = 3000 × 2.40 × 1.50 × 0.5 = 5400.000
暴击1hit   = 5400.000 × 1.50 = 8100.000
期望1hit   = 5400.000 × (1 + 0.30×0.50) = 6210.000
```

### 8.2 多段示例（专3/M3）
```text
倍率: [1.20, 1.80, 2.40]
逐段按同口径计算后求和，输出:
总伤(期望)：...；1hit：...；2hit：...；3hit：...
| 总伤(暴击)：...；1hit：...；2hit：...；3hit：...
| 总伤(不暴击)：...；1hit：...；2hit：...；3hit：...
```

## 9. 实现约定（代码）
- 每个区一个变量，命名建议：`zoneBase`, `zoneCrit`, `zoneDamageBonus`...
- 禁止把多个区揉成一个“临时总加成”变量
- 先统计角色侧区：基础伤害区、暴击区、伤害加成区、伤害减免区、增幅区、连击增伤区、特殊乘区
- 再统计敌方侧区：脆弱区、易伤区、防御区、失衡易伤区、抗性区
- 文本输出和代码计算都按“角色侧 -> 敌方侧 -> 汇总”顺序组织
- 伤害加成区实现时必须拆分：
  - `bonusUnconditional`（无条件）
  - `bonusConditional`（有条件）
  - `bonusFromOthers`（来自其他角色）
  - 并提供 `triggerConditions` 开关集控制有条件项是否生效（默认全 false）
- 统一在末端汇总：

```text
damage_i = zoneBase × zoneCrit × zoneDamageBonus × ... × zoneSpecial
```

## 10. 暂不纳入（后续版本）
- 敌方防御/抗性/减伤的精细模型（当前仅使用防御区=0.5、抗性区=1 的默认化处理）
- 易伤、脆弱、增幅等乘区的更细粒度触发编排与自动判定
- 触发条件型增伤（如“命中后持续 X 秒”）
- 多目标分摊、范围衰减、帧级细节

## 11. 常见错误
- 把百分比直接当整数参与乘法（应先转小数）
- 技能等级取错（默认 9，专3 才用 M3）
- 将“所有伤害加成”重复计入多个乘区
- 多段技能只算首段，未做逐段求和
