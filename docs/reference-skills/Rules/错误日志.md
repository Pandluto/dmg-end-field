## 错误 #1：装备加成理解错误

**优先级**：🔴 高（影响核心计算功能）

**发生时间**：2026-04-05

**错误描述**：
在计算面板攻击力时，错误地将用户提供的"装备加成"理解为"总装备加成（含武器）"，从而扣除了武器贡献。

**错误示例**：
```
用户输入: 装备:力量+166敏捷+279源石技艺强度+69物理伤害加成+59.8%

错误理解:
  敏捷装备加成 = 279 - 156(武器) = 123  ❌

正确理解:
  敏捷装备加成 = 279（其他装备，不含武器）  ✓
  武器贡献由计算器单独计算
```

**错误后果**：
- 面板攻击力计算错误：4522 → 应为 5410
- 误导用户

**根本原因**：
未正确理解 `panel_template.ts` 的设计意图：
- `equipment.mainStat` 参数是"其他装备加成"，不包含武器
- 武器技能贡献由计算器根据 `weaponName` + `matrixCode` 自动计算

**正确做法**：
1. 用户提供的装备加成直接传入 `equipment` 参数
2. 不要手动扣除武器贡献
3. 计算器会自动叠加：角色基础 + 好感 + 武器 + 装备

**正确调用示例**：
```typescript
calculatePanel({
  characterName: '管理员',
  characterLevel: 90,
  weaponName: '宏愿',
  weaponLevel: 90,
  matrixCode: '663',
  potential: 6,
  equipment: {
    mainStat: 279,      // 直接使用用户提供的值
    subStat: 166,
    memoryStrength: 69,
    physicalDmgBonus: 0.598
  }
});
```

**预防措施**：
- 计算/查询类操作优先使用 `src/panel_template.ts` 的 `calculatePanel` 函数
- 装备加成参数不含武器贡献，武器由计算器单独处理
- 如有疑问，先阅读 `src/panel_template.ts` 源码理解参数含义

## 错误 #2：使用错误的 CLI 工具

**优先级**：🟡 中（影响效率）

**发生时间**：2026-04-05

**错误描述**：
在计算面板时使用了 `src/attack_cli.ts`，而非功能更完整的 `src/panel_cli.ts`。

**错误后果**：
- `attack_cli.ts` 不支持装备加成参数
- 无法正确计算含装备的面板

**正确做法**：
- 面板计算使用 `src/panel_cli.ts` 或直接调用 `panel_template.ts`
- `attack_cli.ts` 仅用于简单攻击力计算（无装备场景）

**预防措施**：
- 计算面板时优先使用 `npm run panel` 或 `panel_template.ts`
- 检查 CLI 是否支持所需参数

## 错误 #3：伤害加成“命中判断”口径错误

**优先级**：🔴 高（直接影响伤害结果）

**发生时间**：2026-04-05

**错误描述**：
在判断有条件伤害加成是否计入时，错误地按“技能类别（战技/连携技）是否匹配”来决定，而不是按“该加成能否作用于本次伤害”决定。

**错误示例**：
```
加成条目：寒冷伤害 +16%（触发条件成立）
本次伤害：战技造成寒冷伤害

错误判断：
  因为条目来源写在某个技能分支中，判定不命中  ❌

正确判断：
  本次伤害是寒冷伤害，寒冷伤害加成可直接作用，判定命中  ✓
```

**根本原因**：
把“触发来源”与“生效对象”混淆：
- 触发来源：这条加成在什么条件下被激活
- 生效对象：这条加成最终作用于哪类伤害（元素/类型）

**正确做法**：
1. 先判断触发条件是否满足（已触发/未触发）
2. 再判断加成类型是否能作用于本次伤害（元素/类型命中）
3. 两者都满足才计入本次伤害计算

判定原则（强制）：
`是否命中 = 条件触发状态 × 加成是否可作用于本次伤害`

**预防措施**：
- 在伤害输出中单独列出每条加成的“触发状态”和“命中状态”
- 禁止仅凭“来源技能标签”直接决定是否计入
- 以 `Rules/伤害计算.md` 的“伤害加成区命中口径”为唯一准绳
