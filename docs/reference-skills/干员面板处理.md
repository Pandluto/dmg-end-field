# 干员 max.json 技能数据规范

## 1. 概述

`xxxmax.json` 是干员满配（90级 + 满潜 + 满级技能）数据的精简版本，仅保留 `9`（技能9级）和 `M3`（专三）两级数据，供排轴模拟和伤害计算使用。

---

## 2. 文件位置

```
public/data/characters/
└── {角色名}/
    └── {角色名}max.json
```

---

## 3. 整体结构

```json
{
  "name": "角色名",
  "nameEn": "RomanizedName",
  "rarity": 6,
  "profession": "职业",
  "weapon": "武器",
  "element": "元素属性",
  "mainStat": "主属性",
  "subStat": "副属性",
  "tags": ["标签1", "标签2"],
  "position": ["站位标签"],
  "attributes": {
    "level90": {
      "strength": 158,
      "agility": 95,
      "intelligence": 125,
      "will": 89,
      "atk": 309,
      "hp": 5495
    }
  },
  "talents": [ ... ],
  "potentials": [ ... ],
  "skills": { ... }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 角色中文名 |
| `nameEn` | string | 角色英文名/罗马字 |
| `rarity` | number | 稀有度（4~6） |
| `profession` | string | 职业 |
| `weapon` | string | 武器类型 |
| `element` | string | 元素属性（`ice` / `physical` / `fire` 等） |
| `mainStat` | string | 主属性 |
| `subStat` | string | 副属性 |
| `tags` | string[] | 角色标签列表 |
| `position` | string[] | 推荐站位标签 |
| `attributes.level90` | object | 90级六围属性 |

## 4. Talents（天赋）

### 4.1 结构

```json
"talents": [
  {
    "name": "天赋名",
    "description": "完整描述",
    "{effectField}": 值
  }
]
```

### 4.2 层级处理

- **去层级**：原始数据的 `levels: { breakthrough1: {...}, breakthrough2: {...} }` 压缩为顶层字段
- **取最高突破级**：只取满配（通常为 breakthrough2 或 breakthrough3）的数值
- **防重名**：同名天赋保留两个条目，以数组形式呈现

### 4.3 常见 effectField

| 字段 | 类型 | 说明 |
|------|------|------|
| `energyGain` | number | 终结技能量获得 |
| `extraEnergyOnSelfFreeze` | number | 自身冻结时额外获得终结技能量 |
| `cooldown` | number | 冷却时间（秒） |
| `atkBonus` | number | 攻击力提升比例 |
| `duration` | number | 持续时间（秒） |
| `radius` | number | 范围半径（米） |
| `allySpeedUp` | number | 友方加速比例 |
| `enemySlow` | number | 敌人缓速比例 |
| `lingerDuration` | number | 离场后效果持续时间（秒） |
| `tornadoDamageBonus` | number | 水龙卷伤害提升 |
| `coldVulnerabilityPerConsumedStack` | number | 每消耗寒冷层数提升的寒冷脆弱 |
| `coldVulnerabilityMultiplierInUltimate` | number | 终结技中寒冷脆弱倍率放大 |
| `dotAtkScalePerSec` | number | 每秒持续伤害攻击力比例 |
| `physicalVulnerability` | number | 物理易伤比例 |
| `fireVulnerability` | number | 灼热易伤比例 |
| `extraFireDamageScale` | number | 额外灼热伤害比例 |
| `healByIntelligenceScale` | number | 智力回血比例 |
| `burningBonusMultiplier` | number | 燃烧状态加成倍率 |
| `atkBonusPerStack` | number | 每层攻击力提升 |
| `maxStacks` | number | 最大叠加层数 |
| `extraImbalance` | number | 额外失衡值 |
| `rareScalePer10Intelligence` | number | 每10点智力的稀有提升比例 |
| `maxRareScale` | number | 稀有概率上限 |

```json
  }

```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | number | 潜能编号（1~5） |
| `name` | string | 潜能名称 |
| `description` | string | 描述文字（使用中文直角引号「」） |
| `stats` | object? | 属性能耗计（可选） |

## 6. Skills（技能）

### 6.1 四大技能分类

| 分类 | 英文键名 | 中文名 |
|------|----------|--------|
| 普通攻击 | `normalAttack` | 普通攻击 |
| 战技 | `skill` | 战技 |
| 连携技 | `chainSkill` | 连携技 |
| 终结技 | `ultimate` | 终结技 |

### 6.2 技能基础字段

```json
{
  "name": "技能名",
  "type": "普通攻击",
  "description": "技能完整描述（使用中文直角引号「」）",
  "hitCount": 5,
  "damage": { ... },
  "multipliers": { ... },
  "imbalanceValue": 17,
  "abnormalType": "freeze"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 技能名称 |
| `type` | string | 技能类型 |
| `description` | string | 技能完整描述（中文直角引号「」） |
| `hitCount` | number | 技能段数（普通攻击通常为4~5，其他为1） |
| `damage` | object | 伤害倍率（见 6.3） |
| `multipliers` | object | 完整属性（见 6.4） |
| `imbalanceValue` | number? | 主失衡值 |
| `abnormalType` | string? | 物理异常触发类型（`freeze` / `launch` / `slam` 等） |
| `finalImbalanceValue` | number? | 终结技最后一段的特殊失衡值 |

### 6.3 damage 字段结构

```json
"damage": {
  "9": { "hit1": 0.32, "hit2": 0.18, "hit3": 0.5, "hit4": 0.5, "hit5": 0.99 },
  "M3": { "hit1": 0.39, "hit2": 0.23, "hit3": 0.62, "hit4": 0.62, "hit5": 1.24 }
}
```

- **仅含伤害倍率**：不含 cooldown、energy、recover 等附加字段
- **hit1 ~ hitN**：按段数展开，N = `hitCount`
- **固定两级**：只保留 `"9"`（技能9级）和 `"M3"`（专三）

### 6.4 multipliers 字段结构

```json
"multipliers": {
  "9": { "hit1": 0.32, "hit2": 0.18, ..., "execute": 7.2, "plunge": 1.44 },
  "M3": { "hit1": 0.39, "hit2": 0.23, ..., "execute": 9.0, "plunge": 1.8 }
}
```

- **保留完整属性**：包含 cooldown、energy、damage 在内的所有字段
- **固定两级**：只保留 `"9"` 和 `"M3"`

### 6.5 普通攻击 multiplier 完整字段

```json
"hit1": 0.32,    // 第1段伤害倍率
"hit2": 0.18,    // 第2段
"hit3": 0.5,     // 第3段
"hit4": 0.5,     // 第4段
"hit5": 0.99,    // 第5段
"execute": 7.2,   // 处决攻击倍率
"plunge": 1.44   // 下落攻击倍率
```

### 6.6 战技 multiplier 常见字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `damage` | number | 伤害倍率 |
| `imbalance` | number | 失衡值 |
| `shotDamage` | number | 射击伤害倍率 |
| `shotImbalance` | number | 射击失衡值 |
| `phantomDamage` | number | 幻影追击伤害 |
| `energyRefund` | number | 技力返还 |
| `ultimateEnergyGain` | number | 终结技能量获得 |
| `duration` | number | 持续时间（秒） |
| `recover1Stack` ~ `recover4Stack` | number | 每消耗1~4层附着的技力恢复 |
| `floatDuration` | number | 浮空持续时间 |
| `abnormalType` | string | 异常触发类型 |

### 6.7 连携技 multiplier 常见字段

| 字段 | 类型 | 说明 |
|------|------|------|

| `extraUltimateEnergyPerStack` | number | 每层额外终结技能量 |

### 6.9 多段终结技结构

当终结技包含多段伤害（如多段斩击、戳击+斩击等）时，需按段数展开 `hitCount`、`damage` 和 `multipliers`。

#### 6.9.1 陈千语「冽风霜」示例（7段斩击）

- **描述**：7段斩击，前6段伤害相同，最后一段伤害更高
- **hitCount**：7

```json
"ultimate": {
  "name": "冽风霜",
  "type": "终结技",
  "description": "7段斩击，最后一段伤害更高",
  "hitCount": 7,
  "damage": {
    "9": { "hit1": 0.65, "hit2": 0.65, "hit3": 0.65, "hit4": 0.65, "hit5": 0.65, "hit6": 0.65, "hit7": 8.18 },
    "M3": { "hit1": 0.81, "hit2": 0.81, "hit3": 0.81, "hit4": 0.81, "hit5": 0.81, "hit6": 0.81, "hit7": 10.23 }
  },
  "multipliers": {
    "9": { "hit1": 0.65, "hit2": 0.65, "hit3": 0.65, "hit4": 0.65, "hit5": 0.65, "hit6": 0.65, "hit7": 8.18, "energy": 70 },
    "M3": { "hit1": 0.81, "hit2": 0.81, "hit3": 0.81, "hit4": 0.81, "hit5": 0.81, "hit6": 0.81, "hit7": 10.23, "energy": 70 }
  },
  "imbalanceValue": 15,
  "finalImbalanceValue": 20
}
```

#### 6.9.2 洛茜「利爪」奇袭示例（戳击+斩击）

- **描述**：短时间多段灼热伤害后接两段高伤斩击
- **hitCount**：3（1次戳击 + 2次斩击）
- **特殊字段**：`stabTotal`（戳击总伤害）、`slash1`（第一段斩击）、`slash2`（第二段斩击）

```json
"ultimate": {
  "name": "「利爪」奇袭",
  "type": "终结技",
  "description": "短时间多段灼热伤害后接两段高伤斩击并施加灼热附着，暴击时暴伤更高。",
  "hitCount": 3,
  "damage": {
    "9": { "hit1": 4.75, "hit2": 2.0, "hit3": 6.0 },
    "M3": { "hit1": 6.0, "hit2": 2.5, "hit3": 7.5 }
  },
  "multipliers": {
    "9": { "energy": 110, "stabTotal": 4.75, "slash1": 2.0, "slash2": 6.0, "critDmgBonus": 0.6 },
    "M3": { "energy": 110, "stabTotal": 6.0, "slash1": 2.5, "slash2": 7.5, "critDmgBonus": 0.6 }
  },
  "imbalanceValue": 25
}
```

#### 6.9.3 多段终结技字段规范

| 字段 | 类型 | 说明 |
|------|------|------|
| `hitCount` | number | 技能总段数 |
| `hit1` ~ `hitN` | number | 各段伤害倍率（展开至每一段） |
| `stabTotal` | number | 戳击/穿刺总伤害（部分终结技有） |
| `slash1` | number | 第一段斩击伤害（部分终结技有） |
| `slash2` | number | 第二段斩击伤害（部分终结技有） |
| `finalImbalanceValue` | number? | 最后一段的特殊失衡值 |

## 7. 命名规范

### 7.1 字段命名

- 使用 **camelCase**（如 `hitCount`、`energyGain`）
- 使用 **中文直角引号**「」包裹技能名、终结技名等中文专有名词
- 不使用 ASCII 双引号 `""` 在字符串内部

### 7.2 数值精度

- 伤害倍率：最多2位小数（如 `0.32`、`1.62`）
- 百分比：最多2位小数（如 `0.3` = 30%）
- 能量/技力：整数

---

## 8. JSON 规范

- 文件编码：**UTF-8**
- 使用**中文直角引号**「」替代 ASCII 双引号 `"` 作为字符串内部引号
- 每个 `xxxmax.json` 独立文件，不与原始 `xxx.json` 合并
- 顶级字段顺序：`name` → `nameEn` → `rarity` → `profession` → `weapon` → `element` → `mainStat` → `subStat` → `tags` → `position` → `attributes` → `talents` → `potentials` → `skills`
