# 武器 max 面板处理规范

## 1. 概述

`{武器名}max.json` 是由武器{武器名}.json提取的 面板展示与面板计算使用的精简数据。  
该文件用于配置界面武器模块展示，重点保留：

- 武器基础信息
- 关键攻击力档位（1 / 80 / 90）
- 3 个武器技能在高等级阶段的数值与描述

---

## 2. 文件位置

```
public/data/weapons/
└── {武器名}/
    ├── {武器名}.json
    └── {武器名}max.json
```

示例：

```
public/data/weapons/赫拉芬格/赫拉芬格max.json
```

---

## 3. 顶层结构

```json
{
  "name": "赫拉芬格",
  "rarity": 6,
  "description": "武器描述",
  "attackGrowth": {
    "1": 51,
    "80": 454,
    "90": 505
  },
  "skills": {
    "skill1": { ... },
    "skill2": { ... },
    "skill3": { ... }
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 武器中文名 |
| `rarity` | number | 稀有度 |
| `description` | string | 武器描述 |
| `attackGrowth` | object | 关键等级攻击力（`1`/`80`/`90`） |
| `skills` | object | 三个武器技能 |

---

## 4. attackGrowth 规则

`max` 面板文件只保留关键等级攻击力，不要求 1~90 全量成长表。

```json
"attackGrowth": {
  "1": 51,
  "80": 454,
  "90": 505
}
```

规则：

- key 必须为字符串
- value 必须为数值
- 必须至少包含 `90`

---

## 5. skills 规则

### 5.1 结构

```json
"skills": {
  "skill1": {
    "name": "技能1名",
    "statType": "strength",
    "levels": {
      "7": { "value": 116, "description": "..." },
      "9": { "value": 156, "description": "..." }
    }
  },
  "skill2": {
    "name": "技能2名",
    "statType": "atkPercent",
    "levels": {
      "7": { "value": 0.29, "description": "..." },
      "9": { "value": 0.39, "description": "..." }
    }
  },
  "skill3": {
    "name": "技能3名",
    "statType": "special",
    "levels": {
      "4": {
        "description": "...",
        "passive": { "allSkillDmgBonus": 0.28 }
      },
      "9": {
        "description": "...",
        "passive": { "allSkillDmgBonus": 0.56 }
      }
    }
  }
}
```

### 5.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 技能名称 |
| `statType` | string | 主效果类型（如 `strength` / `atkPercent` / `special`） |
| `levels` | object | 等级字典，key 为等级字符串 |
| `levels.*.value` | number? | 单值技能数值 |
| `levels.*.description` | string | 该等级效果描述 |
| `levels.*.passive` | object? | 无条件生效字段（用于计算） |

### 5.3 passive 字段口径（重要）

- `levels.*.passive` 存的是 **武器 max.json 原始落盘字段**，不是 Buff `type` 表。
- Buff 标准字段只认：
  - [武器buff处理.md](/C:/Users/zsk86/Desktop/dmg/dmg-end-field/docs/reference-skills/武器buff处理.md)
  - [干员buff处理.md](/C:/Users/zsk86/Desktop/dmg/dmg-end-field/docs/reference-skills/干员buff处理.md)
- 面板/持久化字段只认：
  - [storage.ts](/C:/Users/zsk86/Desktop/dmg/dmg-end-field/src/types/storage.ts)
  - [OperatorConfigPanel.tsx](/C:/Users/zsk86/Desktop/dmg/dmg-end-field/src/components/CanvasBoard/components/OperatorConfigPanel.tsx)
- 不要在这份文档里重复定义敌方乘区、增幅区、条件 Buff 的标准 `type`。

示例（当前仓库真实存在的原始写法）：

```json
"passive": {
  "atkPercent": 0.196
}
```

#### C. 当前样本里存在，但不应写成“已接入主链路标准字段”

- `artsDmgBonus`
- `magicDmgBoost`
- `burnDmgBonus`
- `critRate`
- `critDmgBonus`

这些字段目前的正确表述是：

- 可以继续出现在历史 `max.json.passive` 中
- 当前不应被文档写成“标准 Buff type”或“标准面板字段”
- 如果后续要正式接入，先改代码，再改文档

#### D. 不属于当前 `max.json.passive` 面板直连字段

- `allDmgBonus`
- `physicalVulnerability / fireVulnerability / electricVulnerability / iceVulnerability / natureVulnerability`
- `magicTakenDmgBonus`
- `magicVulnerability`

以上内容若存在，应进入 `武器buff处理.md` 管理的 `buff.json` 条件 Buff 体系，而不是写入当前 `max.json.passive`。

## 6. 面板展示映射（配置界面）

武器模块推荐最小展示字段：

- 武器名：`name`
- 稀有度：`rarity`
- 攻击（90级）：`attackGrowth["90"]`
- 技能1/2/3 名称：`skills.skillX.name`
- 当前档位效果：`skills.skillX.levels[{展示等级}]`

展示等级建议：

- skill1 仅使用 `7/9`
- skill2 仅使用 `7/9`
- skill3 仅使用 `4/9`
- 默认展示等级可统一为 `9`
- 若 UI 需要切换，可在 `levels` 中按 key 切换

---

## 7. 数值与文案规范

- 百分比一律用小数表示（`39% -> 0.39`）
- 描述文本保留原语义，不重写数值逻辑
- `passive` 只放“无条件生效”条目
- 条件触发条目写在 `description`，不直接并入面板基础值
- `passive` 优先保留原始字段名；若出现历史别名，允许继续保留，但文档必须在 5.3 中写清当前代码消费状态
