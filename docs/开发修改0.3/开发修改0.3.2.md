## 1. 需求澄清与目标定义

### 1.1 打开后的头像层
- 当配置界面打开后，在配置界面右侧渲染“原技能沙盒中的干员头像集合”。  
- 头像渲染位置沿用技能沙盒原来的位置关系（保持右侧沙盒感知一致）。  
- 头像数量上限与现有规则一致（最多 4 个），来源统一为 `selectedCharacters`。  
- 右侧头像可交互：单击任一头像可切换当前选中干员。

### 1.2 左侧测试文案
- 根据当前选中干员，在配置界面左侧渲染其名字（测试用占位文案）。  
- 名字渲染必须实时跟随切换，不允许延迟状态不同步。

### 1.3 选中指示图形
- 在“当前选中干员头像图层下方”渲染一个亮黄色平行四边形。  
- 图形方向要求：底边在竖直方向。  
- 该平行四边形必须始终存在（配置界面打开且有可选干员时），并随当前选中干员头像位置转移。  
- 单击其他干员后，图形移动到新选中头像下方。

## 3. 技术方案设计

## 3.1 结构层（组件职责）
- 在 `OperatorConfigPanel` 内新增三个语义区域：  
  - `config-avatar-strip`：右侧头像层（最多 4 个，位置对齐沙盒原布局）  
  - `config-selected-name`：左侧当前干员名字（测试用）  
  - `config-selection-indicator`：亮黄色平行四边形指示器  
- 保持 `CanvasBoard` 作为状态来源，`OperatorConfigPanel` 作为纯渲染+交互派发层。

## 3.2 状态层（单一数据源）
- 继续使用 `activeConfigCharacterId` 作为“当前选中干员”唯一依据。  
- 打开配置界面时：若 `activeConfigCharacterId` 为空，自动回退到 `selectedCharacters[0]` 作为默认选中。  
- 在配置界面右侧单击头像时，仅更新 `activeConfigCharacterId`，不新增并行状态。

## 3.3 交互层（切换与同步）
- 右侧头像单击切换选中干员，左侧名字立即同步更新。  
- 指示图形随选中项移动，允许使用 transform 过渡（平滑但不影响布局）。  
- 当头像数量变化（例如重新选人）时，指示图形自动对齐新的选中项；若原选中已不存在，回退首个头像。

## 3.4 指示图形实现（平行四边形）
- 颜色固定亮黄色（建议 `#FFFF00`），但抽为可配变量。  
- 形状建议使用 `transform: skew(...)` 或 `clip-path` 构造，保证“底边在竖直方向”。  
- 定位策略：  
  - 优先方案：基于选中头像容器的 offset 计算指示器 x/y 并绝对定位。  
  - 简化方案：将指示器作为选中项伪元素渲染，依赖项内定位实现“随选中迁移”。  
- 不论方案，图形都需“永远存在于当前选中项下方”。

## 3.5 可读性与注释标准（强制）
- 事件注释：单击切换函数必须说明触发时机、入参、状态影响。  
- 回退逻辑注释：说明为何选择首个头像作为兜底，避免空选中状态。  
- 几何注释：说明平行四边形方向定义与定位计算方式。  
- 样式注释：说明颜色、尺寸、层级阈值来源，避免魔法数字。  
- 文件结构：保持“类型定义区 → 状态推导区 → 事件区 → 渲染区”。
