## 1. 需求澄清与目标定义

### 1.1 触发条件
- 用户在技能沙盒中双击干员头像（`.sandbox-avatar`）时触发配置界面打开。  
- 示例对象包含“别礼”等任意已选干员，行为应统一。

### 1.2 新界面定位
- 新界面名称：**干员面板配置界面**（简称：配置界面）。  
- 显示层级：位于当前页面顶层（覆盖现有三栏内容）。  
- 布局位置：从右侧滑出，占据界面宽度约 **65%**。  
- 垂直尺寸：内容区域占可视高度约 **90%**，在 Y 轴方向居中。  
- 背景色：`#F0F0F0`。
- 内容策略：配置界面主体保持留白，不放置复杂配置项；右侧仅保留一个返回按钮。

### 1.3 关闭方式
- 配置界面内仅提供一个“返回按钮”（黑底白字），点击后关闭。  
- 点击配置界面外侧遮罩区域也可关闭。  
- 关闭时统一执行同一套状态回收逻辑，避免分支不一致。

### 1.4 引导聚焦
- 配置界面外内容添加暗色遮罩，降低背景干扰并引导注意力到抽屉面板。  
- 遮罩与抽屉需要分层清晰，且不遮挡抽屉交互。

- 已存在悬停视觉与角标逻辑，本次不改变其业务行为，仅叠加“双击头像打开配置界面”能力。

## 3. 技术方案设计

## 3.1 结构层（组件与挂载位置）
- 保持 `SkillSandbox` 主体渲染不变，只在头像节点新增 `onDoubleClick` 触发入口。  
- 在 `CanvasBoard` 顶层新增“抽屉 + 遮罩”渲染层，使用条件渲染控制显隐。  
- 配置界面建议独立成小组件（如 `OperatorConfigPanel`），降低主文件复杂度并便于维护。

## 3.2 状态层（可读性优先）
- 新增语义化状态，例如：  
  - `isConfigPanelOpen`：是否打开配置界面  
  - `activeConfigCharacterId`：当前配置目标干员 ID  
- 打开行为：双击头像时写入目标干员并打开。  
- 关闭行为：返回按钮与遮罩点击共用 `closeConfigPanel`，统一清理目标干员状态。

## 3.3 交互层（抽屉与遮罩）
- 抽屉进入：右侧滑入动画（translateX + opacity 过渡）。  
- 遮罩进入：淡入动画（opacity 过渡）。  
- 点击规则：  
  - 点击抽屉内部不触发关闭（阻止冒泡）  
  - 点击遮罩区域触发关闭  
  - 点击返回按钮触发关闭  
- 后续可增强：按 `Esc` 关闭（本计划列为可选增强项）。

## 3.4 样式层（参数化约束）
- 关键参数以 CSS 变量或常量集中配置：  
  - `--config-panel-width: 65vw`  
  - `--config-panel-height: 90vh`  
  - `--config-overlay-bg: rgba(0, 0, 0, 0.45)`  
  - `--config-panel-z: 120`、`--config-overlay-z: 110`  
  - `--config-motion-duration: 220ms`  
- 抽屉定位采用右锚点（`right: 0`），并保持垂直居中。  
- 配置界面内容区默认留白，不铺设表单与列表，只保留必要容器结构。  
- 返回按钮固定语义：黑底白字，且为右侧唯一操作按钮，悬停态仅做轻微反馈。

## 3.5 注释与可读性标准（强制）
- 事件处理函数注释：写清“触发时机 / 输入 / 输出 / 副作用”。  
- 状态流注释：写清“打开与关闭如何收敛到统一路径”。  
- 布局参数注释：解释 65% / 90% / 层级值来源，避免魔法数字。  
- 动画注释：解释为何选 translate 而不是 width 变化（降低重排成本与闪动风险）。  
- 文件组织：保持“类型定义区 → 状态计算区 → 事件处理区 → 渲染区”顺序。
