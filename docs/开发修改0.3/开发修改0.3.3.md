## 1. 需求澄清与目标定义

### 1.1 数据来源
- 在配置面板中，根据当前选中角色读取对应 `*max.json` 数据。  
- 示例：选中“别礼”时读取 `别礼max.json`。  
- 选中角色切换时，面板内容同步切换到新角色数据。

### 1.2 布局总结构（5 模块）
- 总计 5 个模块：上区 4 模块 + 下区 1 条 CTI 入口。  
- 上区可滚动；滚动条隐藏但保留滚动能力。  
- 下区为一条较窄控制台 CTI 入口区域（留白占位）。

### 1.3 上区模块分布
- 上区分左右两列：左列 30%，右列 70%。  
- 左列（30%）分为三块：  
  - 左上：基础数据面板  
  - 左中：技能区（ABEQ）  
  - 左下：武器面板（先留白）  
- 右列（70%）为装备面板（先留白）。

### 1.4 边界约束
- 配置界面内所有模块内容区（文字、卡片、滚动内容、占位块）必须在黄色平行四边形左侧结束。  
- 黄色平行四边形继续作为选中指示器，不参与数据面板排版流。

## 3. 技术方案设计

## 3.1 数据层（按角色映射 max.json）
- 建立“角色名/ID → max.json”映射规则，保证选中即取数。  
- 数据加载建议放在 `OperatorConfigPanel` 内部或其专用 hook，避免污染 `CanvasBoard` 主流程。  
- 增加三态处理：加载中、加载失败、数据为空，防止切换角色时闪烁或空白不可解释。

## 3.2 结构层（配置面板内部）
- 保持当前配置面板容器不变，在 `config-panel-content` 内建立：  
  - `config-top-grid`（上区，可滚动）  
  - `config-cti-strip`（下区，留白）  
- `config-top-grid` 内部放四块：`base-panel`、`skill-abeq-panel`、`weapon-panel`、`equip-panel`。

## 3.3 布局层（30/70 + 边界锁定）
- 列布局使用左 30% / 右 70%。  
- 左列采用三等分（基础 / 技能ABEQ / 武器），右列为单块装备面板，形成上区四模块。  
- 在右侧头像列与黄色指示器之间预留“安全间距变量”，确保所有内容右边界不越过指示器左侧。  
- 若窗口宽度不足，优先收缩面板内部内容，不突破边界约束。

## 3.4 交互层（随选中角色联动）
- 当前选中角色变化时：  
  - 触发重新加载对应 `*max.json`  
  - 更新左上基础数据与左中技能ABEQ展示  
  - 武器/装备面板保留留白占位（显示“待接入”）

## 3.5 滚动层（可滚动 + 隐藏滚动条）
- 上区容器开启纵向滚动。  
- 使用跨浏览器隐藏滚动条方案（WebKit / Firefox / Edge）但不禁用滚动。  
- 下区 CTI 入口固定在底部，不随上区滚动。

## 3.6 注释与可读性标准（强制）
- 数据映射注释：说明角色如何映射到 `*max.json`，异常如何兜底。  
- 布局注释：说明为何采用 30/70、为何设置边界安全间距。  
- 滚动注释：说明“隐藏滚动条但保留滚动”的实现原因。  
- 占位注释：武器/装备留白明确标注“当前阶段不接业务字段”。  
- 文件结构保持“类型区 → 常量区 → 数据区 → 渲染区”。
