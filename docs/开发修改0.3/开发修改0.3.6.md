## 1. 需求目标（0.3.6）

针对“配置界面 -> 装备模块”进行结构化升级，分为两块：

1. 上部分：**装备数值**（70% 高度，表单填写，可实时保存）。  
2. 下部分：**三件套效果**（30% 高度，先留白，预留“非条件触发 / 条件触发”两子区）。  

交付目标：

- 形成稳定可扩展的装备输入区布局与字段定义；
- 每个输入字段与统一 type 一一对应；
- 输入实时写入当前角色配置并落到 sessionStorage；
- 信息快照本期不新增装备文案段落要求，仅保证配置持久化稳定。

## 2. 模块结构设计

## 2.1 装备模块总布局（单模块内部分割）

- 外层仍为“装备”模块卡片；
- 内部分为上下两个方框：
  - 上方：`装备数值` 容器，占比约 70%；
  - 下方：`三件套效果` 容器，占比约 30%。

## 2.2 三件套效果方框（本期留白）

- 顶层标题：`三件套效果`；
- 子分区：
  - `1、非条件触发部分`（留白占位）
  - `2、条件触发部分`（留白占位）
- 本期不接入业务数据，不参与面板计算。

---

## 3. 装备数值表单设计（上部分）

每个字段采用独立小方框样式：`文字 + 可填框`，例如：`力量（+42）`。

## 3.1 行列编排（固定）

- 第 1 行（6 个）：力量、敏捷、智识、意志、主能力、副能力  
- 第 2 行（4 个）：暴击率、暴击伤害、防御值、生命  
- 第 3 行（3 个）：物理伤害加成、灼热伤害加成、电磁伤害加成  
- 第 4 行（3 个）：寒冷伤害加成、自然伤害加成、法术伤害加成  
- 第 5 行（3 个）：战技伤害加成、连携技伤害加成、终结技伤害加成  
- 第 6 行（3 个）：普通攻击伤害加成、对失衡目标伤害加成、源石技艺强度  

## 3.2 输入语义

- 百分比类：输入建议按百分数展示，落盘统一为小数；
- 数值类：输入按整数/浮点数落盘；
- 空值处理：空输入按 0 存储，避免 `undefined` 导致展示与计算分叉。

---

## 4. 统一 type 口径（0.3.6 生效）

以下为本期装备模块采用的目标映射：

- 力量 -> `strength`
- 敏捷 -> `agility`
- 智识 -> `intelligence`
- 意志 -> `will`
- 主能力 -> `mainStatBoost`
- 副能力 -> `subStatBoost`
- 生命 -> `hp`
- 暴击率 -> `critRateBoost`
- 暴击伤害 -> `critDmgBonusBoost`
- 防御值 -> `defense`
- 源石技艺强度 -> `sourceSkillBoost`
- 物理伤害加成 -> `physicalDmgBonus`
- 灼热伤害加成 -> `fireDmgBonus`
- 电磁伤害加成 -> `electricDmgBonus`
- 寒冷伤害加成 -> `iceDmgBonus`
- 自然伤害加成 -> `etherDmgBonus`
- 法术伤害加成 -> `magicDmgBonus`
- 普通攻击伤害加成 -> `normalAttackDmgBonus`
- 战技伤害加成 -> `skillDmgBonus`
- 连携技伤害加成 -> `chainSkillDmgBonus`
- 终结技伤害加成 -> `ultimateDmgBonus`
- 对失衡目标伤害加成 -> `imbalanceDmgBonus`
- 所有技能伤害加成 -> `allSkillDmgBonus`（本期不在表单显示，保留统一口径）
- 攻击力百分比 -> `atkPercentBoost`（本期不在表单显示，保留统一口径）
- 全元素伤害加成 -> `allDmgBonus`（与 `allSkillDmgBonus` 严格区分）

口径边界：

- `allDmgBonus`（全元素）与 `allSkillDmgBonus`（所有技能伤害）不得混用；
- 历史同义字段（如 `artsDmgBonus`、`burnDmgBonus`、`attackPowerPercentBonus`）不再新增写入；
- 若读取到历史字段，后续实现阶段仅做兼容迁移，不在本计划文档新增历史口径。

## 5. 状态与持久化设计

## 5.1 写入目标

- 写入当前角色配置快照（按角色键）；
- 通过既有 `characterConfigMap -> sessionStorage` 链路持久化；
- 切换角色后按角色键回填对应装备输入。

## 5.2 存储时机

- 每次输入变更即写入内存态；
- 内存态变化触发 sessionStorage 同步；
- 不依赖手动“保存”按钮。

## 5.3 本期不做

- 不将三件套条件效果并入默认面板；
- 不在信息快照新增三件套详细文本；
- 不做跨角色批量复制装备配置。
