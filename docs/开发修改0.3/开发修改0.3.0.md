## 1. 项目现状速记（用于避免误改）

### 1.1 组件职责
- `SkillSandbox` 负责渲染干员小沙盒卡片（头像 + 技能按钮网格），并通过 `onMouseDown` 发起拖拽。  
- `CanvasBoard` 作为容器层，负责组合时间轴、画布、技能沙盒三栏布局。  
- 拖拽核心判定在 `useCanvasDrag`，包含吸附、落点、回滚等逻辑，不属于本次改造目标。

### 1.2 样式现状
- `SkillSandbox.css` 中 `.sandbox-character` 目前使用黑色边框，结构清晰，适合做局部增强。  
- 全局设计变量在 `global.css`，包括 `--spacing-*`、`--font-size-*`、`--border-*`、`--bg-*`。  
- 项目已引入 Tailwind（v4），但现有沙盒样式仍以传统 CSS 为主，本次以“可控、可读”优先，采用“局部类 + CSS 变量 + 必要 utility”混合方案。

## 2. 目标交互定义（大胆尝试但边界清晰）

当鼠标悬停到任一 `.sandbox-character`（例如“别礼”或“管理员”）时，统一触发以下视觉行为：

1. 组件整体微缩放：卡片轻微放大（Hover Scale），形成被聚焦感。  
2. 聚焦高亮：提升亮度/对比度或外发光，显著区分当前卡片与非焦点卡片。  
3. 边框重构：移除现有黑色实线边框。  
4. 角标注入：在卡片左上与右下增加“直角角标（类似「」）”。  
5. 角标规范：颜色 `#FFFF00`，不透明；两边等长；长度 = 卡片长边 * `0.34`；线宽 `6px`。  
6. 背景处理：卡片背景改为透明体系（保证可读前提下半透明或全透明）。  
7. 层级提升：悬停卡片上浮到高图层，避免被相邻卡片遮挡。

## 3. 技术方案设计

## 3.1 结构层（DOM 与类名策略）
- 保持现有 `SkillSandbox.tsx` 结构主干不变，避免影响拖拽命中区域。  
- 仅增加语义化状态类，例如：  
  - `.sandbox-character`（基础态）  
  - `.sandbox-character--hoverable`（可悬停增强）  
  - `.sandbox-character--active`（悬停态，可由 `:hover` 或事件态驱动）  
- 不新增深层嵌套 DOM，优先使用伪元素 `::before` / `::after` 绘制角标，减少模板复杂度。

## 3.2 视觉层（角标与高亮）
- 角标实现采用伪元素，避免额外节点：  
  - `::before` 绘制左上角“L”形角标  
  - `::after` 绘制右下角“L”形角标  
- 长度算法：  
  - 采用 CSS 变量 `--corner-len` 管理角长。  
  - 基础策略：`--corner-len: calc(var(--sandbox-card-long-edge) * 0.34)`。  
  - 由于 CSS 无法直接稳定获取“长边”，实现阶段采用“容器观测 + 变量回写”或“按断点预设”的双方案（见 3.4 风险控制）。  
- 线宽固定 `6px`，颜色固定 `#FFFF00`，不透明。

## 3.3 交互层（悬停态与层级）
- 悬停特效采用过渡动画：`transform`、`filter/box-shadow`、`opacity`、`z-index`。  
- 推荐参数（实现阶段可微调）：  
  - `transform: scale(1.02)`  
  - `opacity: 0.92 -> 1`（或保持 0.96 常态）  
  - `z-index: 1 -> 30`（并配合父容器 `position` 与 stacking context 管理）  
- 为避免“悬停抖动”，禁止悬停态改变布局尺寸（只做 transform，不改 width/height/padding）。

## 3.4 尺寸算法与降级策略（关键）
- 目标要求“角标长度为长边 0.34 倍”，属于动态几何计算。  
- 主方案：在实现阶段用 `ResizeObserver` 读卡片 `width/height`，计算长边并写入 `--corner-len`。  
- 降级方案：当浏览器环境或性能预算不允许观察器时，按断点给出固定值（桌面/中屏/小屏三档）。  
- 无论采用哪种方案，代码必须附注释说明“精度 vs 成本”权衡。

## 4. 可读性与注释标准（强制）

后续实现时必须满足：

1. 事件处理器注释：说明触发时机、输入、输出、副作用。  
2. 样式参数注释：说明为何是 `0.34`、`6px`、`z-index 30` 等关键阈值。
