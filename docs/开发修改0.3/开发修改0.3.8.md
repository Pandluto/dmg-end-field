## 目标 DOM 结构

```
.side-panel-body
  └── .tab-content-damage
        ├── .damage-input-section（文本输入区）
        │     └── input.damage-input（文本框）
        └── .damage-display-section（陈列区）
              ├── .refresh-row（刷新行）
              │     └── button.refresh-button（刷新按钮）
              └── .buff-list（Buff 列表容器）
                    └── .buff-item（单个 Buff 行）× N
```

***

## 数据流程

```
用户点击刷新按钮
    ↓
读取 sessionStorage：
  - selectedCharacters（已选角色列表）
  - characterWeapons（角色武器配置映射）
    ↓
构造 fetch 请求列表：
  - /data/characters/{角色名}buff.json
  - /data/weapons/{武器名}/{武器名}buff.json
    ↓
并行加载所有 buff.json
    ↓
合并所有 buff 数据
    ↓
存储到 sessionStorage：allBuffList
    ↓
渲染 Buff 列表到陈列区
```

## 组件结构

### 1. 创建 `src/components/SidePanel/components/DamageTab.tsx`

**用途**："伤害加成"标签页的内容组件

**功能描述**：

- 渲染文本输入框（height 40px，黑框，无填充，水平居中）
- 渲染陈列区（height 固定像素值，占面板剩余空间）
- 渲染刷新按钮（40px × 20px）
- 处理刷新按钮点击事件，加载并陈列 Buff 数据

**数据依赖**：

- `sessionStorage.getItem('selectedCharacters')` - 已选角色列表
- `sessionStorage.getItem('characterWeapons')` - 角色武器配置
- `sessionStorage.setItem('allBuffList', ...)` - 存储合并后的 Buff 列表

**加载的 JSON 文件格式示例**：

```json
{
  "buffs": [
    {
      "name": "攻击力提升",
      "value": 0.15,
      "type": "atk"
    }
  ]
}
```

**错误处理**：

- 任一 JSON 加载失败时，在控制台输出警告，不影响其他数据加载
- sessionStorage 数据不存在时，显示空状态提示

### 2. 样式定义（SidePanel.css 追加）

**文本输入区样式**：

- `.damage-input-section`：竖直排布，有 padding
- `.damage-input`：
  - height: 40px
  - border: 2px solid #000（黑框）
  - background: transparent（无填充）
  - text-align: center（水平居中对齐）

**陈列区样式**：

- `.damage-display-section`：

  - overflow: hidden（无滚轮滑动）
- `.refresh-row`：第一行，包含刷新按钮
- `.refresh-button`：
  - width: 40px
  - height: 20px
  - 黑框白底样式
- `.buff-list`：Buff 列表容器，flex 竖直排列
- `.buff-item`：
  - height: 15px
  - 无框黑字
  - 单行显示，溢出省略号
  - 悬停效果：
    - color: #555555;（文字）
    - background: rgba(255, 255, 0, 0.3)（背景）
