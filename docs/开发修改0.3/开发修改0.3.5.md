## 3. 本次计算口径（执行时必须遵守）

### 3.1 攻击力公式

```text
攻击力 = [(角色基础攻击 + 武器基础攻击) × (1 + 百分比加成) + 固定攻击] × (1 + 能力值加成)
```

### 3.2 主副能力换算

```text
能力值加成 = 主能力 × 0.5% + 副能力 × 0.2%
主能力' = 主能力 × (1 + mainStatBoost)
副能力' = 副能力 × (1 + subStatBoost)
allStatBoost 对主副能力同乘
```

### 3.3 武器计入口径

默认面板必须计入：

- 武器基础攻击；
- `skill1.levels.*.value`（固定属性类直给值）；
- `skill2.levels.*.value`（固定属性类直给值）；
- `skill3.levels.*.passive`（无条件部分）。

默认面板不得计入：

- `buff.json` 中条件触发才生效的条目；
- `skill3` 描述里的条件触发文本。

结构补充说明（按 `宏愿max.json` 实际结构）：

- `skill1/skill2` 没有 `passive` 字段，使用 `levels.*.value` 直接计入；
- `skill3` 才有 `levels.*.passive`，其无条件部分计入默认面板；
- 实现时禁止假设“所有 skill 都有 passive”。

### 3.4 典型对照样例（用于验算）

以“宏愿 0潜”为验算点，后续实现必须能在拆解里体现：

- 敏捷加成；
- 攻击百分比加成；
- 源石技艺强度加成；

并能指出这些加成来自武器无条件 `passive`。

## 4. 信息渲染目标结构（对齐模板）

> 目标不是一次性 100% 复制超长模板，而是分阶段补齐，最终覆盖模板关键区块。

### 4.1 第一阶段（必须完成）

- 角色面板头部（角色名/等级/潜能状态）
- 武器头部（武器名/等级/阶段/珠子/武器潜能状态）
- 4维能力值（力量/敏捷/智识/意志）
- 主副能力换算（原值、加成、作用后）
- 基础攻击段（角色基础 + 武器基础 + 百分比来源）
- 攻击力/暴击/源石技艺强度

### 4.2 第二阶段（本期尽量完成）

- 效率类字段（治疗效率、受治疗、连携冷却、充能、失衡）
- 各类伤害加成字段
- 条件说明区（仅说明，不并入默认面板）

### 4.3 渲染原则

- 信息模块只渲染，不做二次推导；
- 统一由 `panelSnapshot/infoSnapshot` 驱动；
- 字段缺失时显示占位，不得静默吞掉。- 武器条件说明（`buff.json`）必须进入角色 sessionStorage 快照，但默认不进入 `infoSnapshot` 文本渲染。
