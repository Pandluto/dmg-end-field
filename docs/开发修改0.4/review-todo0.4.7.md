# review-todo0.4.7

[任务理解]
- 当前锁定功能已经可用，但锁定按钮的视觉质量不达标，属于明显的“临时样式”。
- 本轮目标不是改功能，而是只优化技能弹窗中锁定按钮的布局和样式，让它看起来像原界面的一部分，而不是后补的小挂件。

[约束]
- 技术约束
  - 只做 UI/交互表现优化，不改锁定逻辑。
  - 不改 `TOGGLE_SKILL_BUTTON_LOCK` 的行为，不改右键删除拦截逻辑。
  - 不改 storage / timeline / AppContext 数据结构。
- 不可破坏部分
  - 锁定按钮的点击可用性必须保留。
  - 已锁定 / 未锁定的状态表达必须比现在更清晰，不能为了好看牺牲可识别性。
  - 现有弹窗内容排版不能被按钮挤乱。
- 风格/架构要求
  - 不允许继续把锁定按钮直接混排在 `h4` 文本标题里。
  - 新样式要贴合现有弹窗“紧凑、工具面板、轻工业感”的视觉方向。
  - 优先使用文字 + 小图标或状态胶囊样式，不要只剩一个裸 emoji 按钮。
