# UI 修改 0.2.1

## 修改日期
2026-04-07

---

## 1. 设计概念

**信息面板载体 —— 黄金比例倾斜平行四边形**

引入一个倾斜的圆角平行四边形作为「倒数第二图层」，位于背景层之上、所有内容层之下，承载侧边面板或信息展示功能。利用黄金分割点对齐创造视觉愉悦感，黄色高饱和度作为功能区块的视觉锚点。

---

## 2. 几何定义

### 基础参数
| 项目 | 参数 |
|------|------|
| 宽度 | 360px |
| 高度 | 600px |
| 倾斜方向 | 顶边向左倾斜（逆时针 20°） |
| 圆角 | 适度圆角（建议 8px-12px） |

### 对齐规则
| 项目 | 参数 |
|------|------|
| 对齐点 | 屏幕顶点左黄金分割点（φ ≈ 0.382） |
| 对齐位置 | 顶边中点 |

### 黄金分割计算
```
屏幕宽度（假设 1920px）：
黄金分割点 = 1920 × 0.382 ≈ 733px（从左）

顶边中点 X = 733px
面板中心 X = 733px
面板左侧 X = 733 - 180 = 553px
```

---

## 3. 视觉规范

### 颜色
| 项目 | 色值 | 说明 |
|------|------|------|
| 面板背景 | `#FFFF00` | 高饱和度黄色，功能区块标识 |
| 透明度 | `opacity: 0.15` 或使用 rgba | 避免遮挡内容 |

### 层级关系
```
z-index: 0   →  背景层 (#F0F0F0)
z-index: 1   →  倾斜平行四边形层（当前修改）
z-index: 2+  →  内容层（谱线、技能按钮等）
```

### CSS 实现参考

```css
.skew-panel {
  position: absolute;
  top: 0;
  left: 0;
  width: 360px;
  height: 600px;
  background: rgba(255, 255, 0, 0.15);
  transform: skewX(-20deg);
  border-radius: 10px;
  z-index: 1;
  pointer-events: none;
}
```

## 5. 设计原则

1. **黄金比例对齐** - 顶边中点与黄金分割点对齐，创造自然舒适的视觉位置
2. **低饱和叠加** - 黄色以半透明形式叠加，不抢夺内容焦点
3. **倾斜创造动感** - -20° 倾斜打破直角网格的沉闷感
4. **内容层穿透** - `pointer-events: none` 确保不阻挡下方交互

---

## 6. 预期效果

- 画面左侧出现一个倾斜的黄色半透明平行四边形
- 作为功能区域的视觉暗示，但不喧宾夺主
- 与右侧谱面形成对角线构图，增加设计感
- 整体风格：**年轻、游戏化、节奏感强**

---

## 7. 注意事项

- 黄金分割点计算需根据实际视口宽度动态调整
- 倾斜角度建议 15°-25°，过大会导致内容阅读困难
- 透明度建议 0.1-0.2，确保层叠可读性
- 需测试不同屏幕宽度下的显示效果
