# 草稿：当 AI 进入 Workbench

> 本文是开发手记第 08 篇的写作草稿，不参与本地阅读页导航。正式内容以 `../08-workbench-state-machine.md` 为准。

Workbench 接入 AI 后，页面、SQLite 和 AI 会话会同时保存“当前状态”。这不是同一类数据：SQLite 的当前正式节点是正式依据，React 页面工作副本负责快速显示和编辑，AI 会话归属限制 AI 能访问哪条正式数据轴。

当三者没有收敛时，不能通过放宽工具权限解决。正确顺序是：读取当前正式节点，把它重新加载到页面，确认页面和节点一致，再允许 AI 会话读取或创建同树候选分支。

本篇需要把几个通用技术概念落到项目中：

- `state machine（状态机）`：进入 AI、工具执行、超时核对和恢复的合法阶段；
- `token`：内部接口、会话归属和用户批准的不同通行证；
- `cache invalidation（缓存失效）`：页面工作副本在 timeline 或 checkout 改变时必须放弃旧内容；
- `SSE`：命令状态的推送渠道，不是成功证明；
- `reconciliation（后续核对）`：超时或中断后，按精确 command ID 查清命令究竟有没有产生副作用；
- `throughput（吞吐）` 与 `sliding window（滑动窗口）`：防止同一会话、同一数据轴的重复请求和迟到命令相互撞车。

写作原则：通用技术名词首次保留英文并给中文说明；DEF 项目概念以中文为主，在括号中给出代码或英文名，并说明它是什么。配置页面也应遵循同一规则：业务对象与技术设置分区显示。
