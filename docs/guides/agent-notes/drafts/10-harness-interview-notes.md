## 第一轮：PSM（问题求解方法）为什么出现在后半篇

### 用户原话

后半篇什么意思呢？就是我先回答你第一个问题。这是一篇开发手记，终将要把概念型的问题落到工程上的问题。PSM 这个问题解决模型。是知识工程的一个特点。但是它跟我们开发垂直领域的智能体有很紧密的联系性。读者不会开发智能体。但读者可能都是做题家。虽然都是概念，但是 PSM 相当于直接讲 harness 更好让读者理解。PSM 讲的是什么呢？一个问题、一个 task 怎么去解决，要怎么做？而智能体它实际是怎么运行的呢？分析和理解读者发出来的对话内容。调用需要的 tools。得到结果，再进行加工，退出循环得到答案。而 harness 的体系呢，作为这一套的润滑者和管理者。而问题解决模型呢可能是个论证证明题。我们要用调用哪些证明方法去加工？怎么一个链路？最后得到什么结论？

### 暂时整理

- 这是一篇**开发手记**。后半篇必须把前面的概念讨论落到工程问题，而不是继续增加概念。
- 读者未必开发过智能体，但熟悉“解题”。因此，PSM 是理解 Harness 的**认知桥梁**。
- PSM 先回答：一个问题或 Task（任务）要怎样解决，需要采用哪些方法，经过怎样的加工链路，最后得到什么结论。
- 智能体的基本运行过程是：理解用户对话，调用所需 Tools（工具），取得结果，继续加工，直到退出循环并给出答案。
- Harness 不是这套解题方法本身，而是围绕这套运行过程发挥**润滑和管理作用**。
- “论证证明题”是当前最重要的通俗类比：需要选择证明方法，组织推导链路，并得到最终结论。
