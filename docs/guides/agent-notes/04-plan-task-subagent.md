# Plan、Task 与 Subagent

Plan、Todo、Task 和 Subagent 经常一起出现，但它们不是四种写法不同的任务列表。

**Plan** 是对步骤和依赖的安排。它帮助 Agent 在工作过程中记住先后关系，也让用户知道现在进行到哪里。Plan 本身不执行任何事情。

**Todo** 更接近状态清单：哪些还没开始，哪些正在处理，哪些已经完成。它可以是 Plan 的外在记录，但不一定包含完整推理。

**Task** 是一块可以交付的工作。它需要明确输入、边界和期望结果。一个 Task 可以由当前 Agent 自己执行，也可以交给另一个 Agent。

**Subagent** 则是拥有独立上下文的执行者。它适合处理边界清楚、可以并行或会产生大量中间信息的工作。父 Agent 通常只需要拿回结论，不必把子 Agent 的完整消息历史塞进自己的上下文。

## 隔离不等于无限授权

Subagent 的价值之一是 Context Isolation，但上下文隔离不应该顺带扩大权限。子 Agent 能使用哪些工具、是否允许继续创建子 Agent、结果以什么格式返回，都应该由父任务和运行时明确限制。

也不是任务一多就应该派生 Subagent。工作很短、强依赖当前上下文，或者多个步骤会频繁修改同一状态时，拆出去反而增加同步成本。先有清楚的任务边界，再谈分工。

是否开放 Plan、Task 或 Subagent，是产品选择，不是 Agent 的必备认证。一个工具循环简单但边界清楚的 Agent，仍然是 Agent；功能很多却无法管理状态和权限，也不会因此更可靠。
