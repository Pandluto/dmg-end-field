# Plan、Task 与 Subagent

“换一件武器”不需要复杂规划。把任务扩大成“检查整支队伍的配装，找出冲突并给出修改建议”以后，Plan、Todo、Task 和 Subagent 才开始有实际差别。

| 名称 | 在这个任务里是什么 |
|---|---|
| Plan | 先读取全队配置，再逐人检查冲突，最后汇总建议。它描述顺序和依赖，不负责执行。 |
| Todo | “读取完成、检查中、待汇总”这样的状态清单。 |
| Task | “检查角色 A 的装备冲突”这一块可以单独交付的工作。 |
| Subagent | 接收角色 A 的配置和检查规则，在单独上下文中完成这项 Task 的执行者。 |

Subagent 的上下文是单独的，但并非凭空开始。父 Agent 会把筛选后的任务材料传进去，再拿回结论；没有必要把子 Agent 的完整消息历史塞回主上下文。

## 隔离不等于无限授权

Subagent 的价值之一是 Context Isolation，但上下文隔离不应该顺带扩大权限。子 Agent 能使用哪些工具、是否允许继续创建子 Agent、结果以什么格式返回，都应该由父任务和运行时明确限制。

任务一多不等于必须派生 Subagent。工作很短、强依赖当前上下文，或者多个步骤会频繁修改同一状态时，拆出去反而增加同步成本。先有清楚的 Task 边界，再谈分工。

是否开放 Plan、Task 或 Subagent，是产品选择。DEF 当前收窄了这组能力，主要执行路径仍是单 Session 的工具循环。
