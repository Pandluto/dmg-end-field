# Plan、Task 与 Subagent

换一件武器，
不需要复杂规划。

把任务扩大一些：

> 检查全队配装，找出冲突。

几个概念开始有区别：

| 名称 | 在这个任务里是什么 |
|---|---|
| Plan | 先读取，再逐人检查，最后汇总 |
| Todo | 读取完成、检查中、待汇总 |
| Task | 检查角色 A 的装备冲突 |
| Subagent | 单独执行这项 Task 的 Agent |

Plan 描述顺序和依赖。

Todo 记录当前状态。

Task 是可以交付的工作。

Subagent 是执行者。

## 单独上下文，不是空白上下文

父 Agent 会传入必要材料：

- 角色 A 的配置；
- 本次检查规则；
- 期望的返回格式。

Subagent 完成后，
只返回结论。

完整消息历史不用塞回主上下文。

## 隔离不等于扩权

子 Agent 的权限要单独限制：

- 能使用哪些工具；
- 能否继续创建 Agent；
- 结果怎样返回。

任务多，
不等于必须派生 Subagent。

以下情况不适合拆：

- 工作很短；
- 强依赖主上下文；
- 多步频繁修改同一状态。

先划清 Task 边界，
再决定是否分工。

是否开放这些能力，
取决于产品选择。
