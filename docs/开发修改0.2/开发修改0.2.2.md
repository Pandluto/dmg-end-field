# 开发修改 0.2.2 - 修复节点吸附 Y 坐标传递错误

## 约束
- 本计划仅针对 0.2.1 重构后节点吸附功能失效的 bug 修复
- 不修改 `snapToNearestNode` 函数逻辑
- 不修改 `findNearestLine` 函数逻辑
- 仅修复 `useCanvasDrag.ts` 中调用 `snapToNearestNode` 时 Y 坐标参数传递错误

## 问题分析

### Bug：节点吸附功能失效

**表现：**
拖拽技能按钮到谱线时，按钮吸附位置不正确或吸附到错误节点。

**根因分析：**

在 `hooks/useCanvasDrag.ts` 的 `handleMouseUp` 中，调用 `snapToNearestNode` 时传入的 Y 坐标错误：

```typescript
// 错误代码 (重构后)
const { staffIndex, lineIndex } = nearestLine;
const mouseX = e.clientX - canvasRect.left;
const mouseY = e.clientY - canvasRect.top;  // ❌ 错误

const { snappedPosition } = snapToNearestNode(
  { x: mouseX, y: mouseY },  // ❌ 传入 mouseY
  ...
);
```

`mouseY` 是鼠标相对画布的 Y 坐标，不包含 `groupOffset`。

但 `snapToNearestNode` 内部会计算 `lineY = calculateLineY(...) + staffHeight/2` 然后返回 `lineY + groupOffset`。

而 `findNearestLine` 返回的 `lineY` 已经包含了 `groupOffset`。

**原始代码正确逻辑：**

```typescript
// 原始代码 (CanvasBoard/index.tsx)
const { staffIndex, lineIndex, lineY } = nearestLine;  // lineY 已含 groupOffset
const mouseX = e.clientX - canvasRect.left;

const { snappedPosition } = snapToNearestNode(
  { x: mouseX, y: lineY },  // ✅ 传入 lineY
  ...
);
```

`findNearestLine` 返回的 `lineY` 包含 `groupOffset`，传给 `snapToNearestNode` 后，虽然内部又加了一次 `groupOffset`（重复计算），但由于 `position.y`（即 `lineY`）已经包含了正确的 Y 坐标，结果是正确的。

## 技术方案

### 修复 `useCanvasDrag.ts` 中的调用

修改 `handleMouseUp` 中 `snapToNearestNode` 的调用，使用 `findNearestLine` 返回的 `lineY`：

```typescript
// 修复后
if (nearestLine) {
  const { staffIndex, lineIndex, lineY } = nearestLine;
  const mouseX = e.clientX - canvasRect.left;

  const { snappedPosition } = snapToNearestNode(
    { x: mouseX, y: lineY },  // ✅ 传入 lineY
    config,
    staffIndex,
    lineIndex,
    canvasWidth,
    skillButtonsRef.current,
    draggingState.characterId
  );
}
```
