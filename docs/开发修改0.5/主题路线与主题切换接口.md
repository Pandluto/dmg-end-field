# 主题路线与主题切换接口

## [阶段结论]

- 当前采用的主主题方向是 `Office 365 Excel`：浅灰窗口背景、白色卡片、Excel 绿语义强调、直角控件、轻量边框、少阴影。
- 本阶段已经完成的是“视觉覆盖层”，不是完整主题系统。
- 后续如果开放主题切换，必须先把主题 token、结构样式、组件状态样式分离，否则继续追加 CSS 会让覆盖顺序失控。

## [当前主题路线]

### 1. 品牌色

当前 Excel 主题使用以下色板：

| 语义 | HEX | 用途 |
| --- | --- | --- |
| `--excel-green-deep` | `#185C37` | 深绿，关键文字、强强调 |
| `--excel-green-main` | `#217346` | 主品牌绿，选中态、按钮、下划线 |
| `--excel-green-bright` | `#50C878` | 亮绿，辅助强调 |
| `--excel-green-lime` | `#B4E67E` | 黄绿，技能类型和轻强调 |
| `--excel-white` | `#FFFFFF` | 卡片、弹窗、输入区 |

### 2. Office 区域色

| 语义 | HEX | 当前映射 |
| --- | --- | --- |
| `--office-window-bg` | `#ECEFF4` | 最外层窗口背景 |
| `--office-content-bg` | `#F3F3F3` | 主内容区背景 |
| `--office-nav-bg` | `#E8EBF0` | 顶部栏、标签栏、导航区 |
| `--office-card-bg` | `#FFFFFF` | 卡片、面板、弹窗 |
| `--office-title` | `#1A1A1A` | 标题文字 |
| `--office-body` | `#333333` | 正文、标签、普通按钮文字 |
| `--office-border` | `#D1D1D1` | 边框、分割线 |
| `--office-placeholder` | `#8A8A8A` | placeholder、弱提示 |

### 3. 设计语言

- 扁平化：去掉重渐变、重阴影、复杂拟物。
- 层次化：窗口背景、内容背景、卡片背景、导航背景分层。
- 语义化色彩：绿色只用于选中、确认、关键状态，不铺满大面积背景。
- 直角优先：工作台、配置面板、按钮、输入框使用直角；头像类元素允许圆形。
- 交互反馈：hover 使用浅绿背景或左侧绿线，不使用黄色大块高亮。

## [当前代码结构]

### 1. Token 入口

当前主题 token 主要堆叠在：

- `src/styles/global.css`

现状问题：

- 文件中同时保留了早期 `Excel Acrylic theme` 和后续 `Office 365 Excel theme` 的 token。
- 最终生效依赖 CSS 顺序，而不是清晰的主题入口。
- `--ide-*`、`--excel-*`、`--office-*` 三组变量混用，短期可用，长期不利于主题切换。

### 2. 组件覆盖层

当前主题覆盖分散在以下文件尾部：

- `src/components/WorkbenchFrame/WorkbenchFrame.css`
- `src/components/CanvasBoard/CanvasBoard.css`
- `src/components/ToolPanel/ToolPanel.css`
- `src/components/SelectionPanel/SelectionPanel.css`
- `src/components/CanvasBoard/SkillButton.css`
- `src/components/CanvasBoard/SkillSandbox.css`

现状问题：

- 多轮覆盖叠加，存在 “final overrides” 依赖顺序。
- 部分例外规则必须放在文件末尾，例如：
  - `config-avatar-item.is-active::after` 平行四边形选中态。
  - `config-data-item.is-main` 主能力黄色背景。
  - `config-avatar-image` 圆形头像。
  - `config-skill-level-switch` 与 `config-weapon-potential-switch` 统一开关样式。
- 后续继续追加主题时，容易误伤这些例外规则。

## [主题系统目标结构]

## [组件样式分层规则]

### 1. 结构样式

结构样式不允许跟主题耦合。

包括：

- `display`
- `position`
- `grid`
- `flex`
- `width`
- `height`
- `overflow`
- `z-index`
- 拖拽命中区域
- canvas / grid / skill button 坐标语义

这些规则继续留在组件 CSS 中，不进入主题文件。

### 2. 主题样式

主题样式必须只使用 token。

包括：

- `background`
- `color`
- `border-color`
- `box-shadow`
- `border-radius`
- `hover / active / selected`
- `focus ring`

后续迁移目标：组件 CSS 中不再出现硬编码色值，除业务特殊色外。

### 3. 业务例外样式

以下样式不是普通主题样式，必须显式保留：

- `config-avatar-item.is-active::after`
  - 角色配置头像选中态平行四边形。
  - 不允许被头像圆形规则裁剪。
  - `.config-avatar-item` 必须 `overflow: visible`。
- `config-data-item.is-main`
  - 主能力固定黄色背景 `#FFF59D`。
  - 不随主题切换变成普通 hover 色。
- `config-data-item.is-sub`
  - 副能力固定灰色背景 `#D9D9D9`。
- `skew-panel / skew-panel-bottom`
  - 背景平行四边形装饰层。
  - 不属于主题切换接口第一阶段，不要删除。
- `config-skill-level-switch` 与 `config-weapon-potential-switch`
  - 必须保持同一视觉规格。

## [主题切换接口预留]

### 1. 推荐入口

使用根节点属性控制主题：

```ts
document.documentElement.dataset.theme = themeId;
```

推荐主题 id：

```ts
type ThemeId =
  | 'office-excel'
  | 'excel-acrylic'
  | 'legacy-black';
```

### 2. 推荐缓存 key

主题偏好是 UI 偏好，不属于 timeline / skill-button / buff-list 主链路。

推荐独立 key：

```ts
ddd.ui.theme.v1
```

value：

```ts
{
  "themeId": "office-excel"
}
```

不要写入：

- `ddd.timeline.data.v1`
- `ddd.skill-button.v1`
- `ddd.all-buff-list.v1`
- `ddd.candidate-buff-list.v1`

### 3. 推荐代码结构

```text
src/core/repositories/themeRepository.ts
src/core/services/themeService.ts
src/hooks/useTheme.ts
src/styles/themes/theme-contract.css
src/styles/themes/theme-office-excel.css
src/styles/themes/theme-excel-acrylic.css
```

职责：

- `themeRepository`
  - 只读写 `ddd.ui.theme.v1`。
- `themeService`
  - 校验 themeId。
  - 提供默认主题。
  - 设置 `document.documentElement.dataset.theme`。
- `useTheme`
  - React 层使用。
  - 负责初始化、切换、暴露当前主题。

## [后续执行顺序]

### Step 1：整理 token

- 新建 `src/styles/themes/theme-contract.css`。
- 把当前 `global.css` 中最终生效的 Office Excel token 迁移进去。
- 保留兼容别名层，避免一次性改所有组件。

### Step 2：拆出主题文件

- 新建 `theme-office-excel.css`。
- 把 `--office-*`、`--excel-*`、`--theme-*` 具体值放进去。
- `global.css` 只保留基础 reset、字体、通用 button/input 基线。

### Step 3：组件 CSS 去硬编码

优先处理：

- `WorkbenchFrame.css`
- `CanvasBoard.css`
- `ToolPanel.css`
- `SelectionPanel.css`

处理原则：

- 只替换颜色、边框、阴影、圆角。
- 不碰布局、拖拽、canvas 坐标、grid 尺寸。

### Step 4：开放切换接口

- 新增 `themeRepository.ts`。
- 新增 `themeService.ts`。
- 新增 `useTheme.ts`。
- 在应用初始化时读取 `ddd.ui.theme.v1`。
- 设置 `document.documentElement.dataset.theme`。

## [不要动]

- 不要改 `timeline.data`、`skill-button`、`buff-list`、`candidate-buff-list` 数据结构。
- 不要改伤害公式、buff 计算、拖拽坐标、grid 吸附逻辑。
- 不要删除背景平行四边形装饰层。
- 不要把角色配置头像选中平行四边形改成圆形。
- 不要把主能力黄色背景改成主题 hover 色。
- 不要把主题偏好写入业务缓存主表。

## [验收标准 AC]

- `npm run build` 通过。
- 默认主题为 `office-excel`。
- 切换主题只改变视觉，不影响：
  - 选人。
  - 进入排轴。
  - 技能按钮拖放。
  - Buff 单击添加。
  - Buff 拖拽添加。
  - 技能弹窗打开。
  - 角色配置面板打开。
- 刷新页面后主题偏好从 `ddd.ui.theme.v1` 恢复。
- 角色配置头像选中态平行四边形仍存在。
- 主能力黄色背景仍存在。

## [给 Trae 的执行指令]

1. 先不要继续追加主题覆盖块。
2. 先完成 token 拆分和兼容别名层。
3. 再逐个组件替换硬编码颜色。
4. 最后再开放 `ddd.ui.theme.v1` 和 `useTheme`。
5. 每一步都必须构建通过，并手测主交互链路不回退。
