## [遗留问题 1]

### useTimelineData 仍直接写 timeline storage

文件：

- `src/hooks/useTimelineData.ts`

现状：

- `useTimelineData.ts` 仍直接 import `STORAGE_KEYS`。
- `useTimelineData.ts` 仍直接 import `setStorageJson`。
- autosave 和 normalize 后保存仍直接写 `STORAGE_KEYS.TIMELINE_DATA`。

问题：

- 这不影响当前功能手测。
- 但不符合 0.5.1 的分层目标。
- Hook 层仍持有 storage key 和写入实现，说明 timeline 保存边界没有完全下沉到 `timelineService` / `timelineRepository`。

暂缓原因：

- 当前没有观察到功能回归。
- 当前主目标是继续推进 0.5.2 / 0.5.3。
- 该问题属于层级边界收口，不是主链路阻塞 bug。

后续修复要求：

- `useTimelineData.ts` 移除 `STORAGE_KEYS` import。
- `useTimelineData.ts` 移除 `setStorageJson` import。
- autosave 统一调用 `saveTimelineDataService(dataToSave)`。
- normalize 后保存也统一调用 `saveTimelineDataService(normalized)`。
- Hook 只保留 React state、ref、debounce、调用 service，不直接写 storage。

触发修复时机：

- 后续修改 `useTimelineData.ts` 保存/加载逻辑时必须补修。
- 后续修改 `timelineService.ts` 或 `timelineRepository.ts` 时必须补修。
- 进入更严格的 service/repository 分层验收前必须补修。

回归要求：

- 新增按钮后刷新页面，timeline 能恢复。
- 移动按钮后刷新页面，位置能恢复。
- 跨干员移动按钮后刷新页面，归属能恢复。
- 删除按钮后刷新页面，删除状态能保持。
- `npm run build` 通过。

## [遗留问题 2]

### 候选 Buff 被强转成已选 Buff 实体类型

文件：

- `src/components/SidePanel/components/DamageTab.tsx`
- `src/core/repositories/candidateBuffRepository.ts`

现状：

- `candidateBuffRepository` 当前使用 `SkillButtonBuff[]` 作为候选 Buff 列表类型。
- `DamageTab.tsx` 中存在类似 `buffs as unknown as SkillButtonBuff[]` 的强转。

问题：

- 候选 Buff 不是已选 Buff 实体。
- 候选 Buff 不应要求稳定实体 `id`。
- `SkillButtonBuff` 属于已选 Buff 实体表 `def.all-buff-list.v1` 的数据类型。
- candidate 类型和 selected entity 类型混用，会增加后续再次污染 `def.all-buff-list.v1` 的风险。

暂缓原因：

- 当前候选 Buff 写入的是 `def.candidate-buff-list.v1`，没有覆盖 `def.all-buff-list.v1`。
- 当前点击添加、拖拽添加、关闭重开已选 Buff 的主链路手测通过。
- 该问题属于类型边界收口，不是当前功能阻塞 bug。

后续修复要求：

- 新增候选 Buff 专用类型，例如 `CandidateBuff`。
- `candidateBuffRepository.getCandidateBuffList()` 返回 `CandidateBuff[]`。
- `candidateBuffRepository.setCandidateBuffList()` 接收 `CandidateBuff[]`。
- `DamageTab.tsx` 使用 `setCandidateBuffList(buffs)`，不得再使用 `as unknown as SkillButtonBuff[]`。
- 已选 Buff 写入仍由 `buffService` / `addSkillButtonBuff` 负责转换成 `SkillButtonBuff`。

触发修复时机：

- 后续执行 0.5.3，抽离 `useCandidateBuffs` 时必须补修。
- 后续修改 `candidateBuffRepository.ts` 时必须补修。
- 后续修改候选 Buff 数据结构、搜索、刷新逻辑时必须补修。

回归要求：

- 刷新候选 Buff 后，候选列表正常展示。
- 搜索候选 Buff 正常。
- 点击候选 Buff 添加到已选 Buff 正常。
- 拖拽候选 Buff 添加到已选 Buff 正常。
- `def.candidate-buff-list.v1` 只保存候选 Buff。
- `def.all-buff-list.v1` 只保存已选 Buff 实体。
- 刷新候选 Buff 不得覆盖 `def.all-buff-list.v1`。
- `npm run build` 通过。
