# 增幅区遗留开发

## 1. 当前讨论结论

- 本文档只记录“增幅区”遗留开发的已确认口径。
- 不处理 UI 主题、交互优化、旧 review-todo 中的暂停项。
- 当前以代码实现方向为主，不再以旧文档是否过时作为阻塞。

## 2. 已确认口径

### 2.1 增幅区定位

- `增幅区` 是独立乘区。
- 它和 `脆弱区 / 易伤区` 同级。
- 它不是：
  - `伤害加成区`
  - `multiplierBonus`
  - `multiplierMultiplier`
  - `skillMultiplierBonus`

### 2.2 当前阶段驱动方式

- 当前阶段，`脆弱区 / 易伤区 / 增幅区` 都只由 `Buff` 驱动。
- 当前阶段不接入：
  - `EquipmentConfig`
  - `equipmentParser.ts`
  - `OperatorConfigPanel.tsx`
  - `infoSnap`
  - `panelSnapshot`

### 2.3 增幅区字段命名

- `physicalAmplify`
- `magicAmplify`
- `fireAmplify`
- `electricAmplify`
- `iceAmplify`
- `natureAmplify`

### 2.4 增幅区取值逻辑

- `physical` 伤害：
  - 只吃 `physicalAmplify`
- 元素伤害：
  - 吃对应元素增幅：
    - `fireAmplify`
    - `electricAmplify`
    - `iceAmplify`
    - `natureAmplify`
  - 再叠加 `magicAmplify`

说明：

- `magicAmplify` 的逻辑与当前 `法术脆弱` 口径一致。
- 也就是“元素伤害 = 对应元素增幅 + 法术增幅”。

### 2.5 增幅区叠加规则

- 多个增幅 Buff 直接相加。
- 最终统一按：

```text
× (1 + amplifyRate)
```

进入总公式。

## 3. 代码实现方向

当前代码里，增幅区应完全仿照 `脆弱区 / 易伤区` 链路实现，而不是仿照 `伤害加成区`。

### 3.1 正确链路

```text
Buff JSON / CandidateBuff.type
-> DamageTab.tsx 添加到按钮
-> useSkillButtonBuffs.ts 持久化已选 Buff
-> buffCalculator.ts / calculateBuffTotals() 汇总增幅字段
-> buffCalculator.ts 提供 calculateAmplifyRate()
-> skillButtonDamageCalculator.ts 把 amplifyRate 接入总公式
-> SkillButton.tsx 展示增幅区总值与分项
```

### 3.2 需要新增的主链路字段

在 `src/core/calculators/buffCalculator.ts` 的 `BuffCalculationResult` 中新增：

- `physicalAmplify`
- `magicAmplify`
- `fireAmplify`
- `electricAmplify`
- `iceAmplify`
- `natureAmplify`

在 `calculateBuffTotals()` 中新增对应 `switch(buff.type)` case。

新增 helper：

- `calculateAmplifyRate(characterElement, buffTotals)`

在 `src/core/calculators/skillButtonDamageCalculator.ts` 中新增：

- `amplifyRate`
- `DamageBreakdown.afterAmplify`
- `SkillButtonDamageResult.amplifyRate`

在 `calculateHitDamage()` 中插入：

```text
afterDefense
-> afterAmplify
-> afterFragile
-> afterVulnerability
```

在 `src/components/CanvasBoard/SkillButton.tsx` 中新增：

- `【增幅区】`
- 增幅区总值
- 各元素/法术分项
- 逐 hit 公式行里的 `× (1 + amplifyRate)`
