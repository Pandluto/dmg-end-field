# 项目规则

## 代码定位

当用户说「定位XXX代码」或「找到XXX在哪个文件」时：

1. 使用 Grep/Read 工具搜索定位
2. 找到后生成可点击的 Markdown 链接：
   ```
   [文件说明](file:///完整绝对路径#L起始行-L结束行)
   ```
3. 同时在回复中简要说明代码作用

## 代码修改

当用户说「修改XXX」时：
1. 先读取相关文件确认当前代码
2. 使用 SearchReplace 工具精确修改
3. 完成后运行 `npm run build` 验证

## 输出格式

每次代码修改后必须包含：
- [变更摘要]：修改了哪些文件，做了哪些改动
- [自检（对照 AC）]：验收标准检查结果
- [遗留问题]：未解决的问题（如有）
