# 武器面板选择交互 Spec

## Why
当前配置界面缺少可视化的武器选择入口，无法在同一交互流程中完成“无/具体武器”切换。需要补充直观、可悬停反馈、可点击抽屉选择的武器配置能力。

## What Changes
- 在“配置界面/武器面板”新增一个武器选择方框，默认值为“无”。
- 方框支持悬停高亮：`rgba(255, 255, 0, 0.3)`。
- 点击方框后，打开“向上抽屉”武器选择列表。
- 抽屉列表数据源为 `public/data/weapons/weapons-list.json`。
- 抽屉内每个选项支持悬停高亮：`rgba(255, 255, 0, 0.3)`。
- 列表包含“无”选项，并允许用户主动选择“无”。
- 选择完成后，方框展示当前武器名，并将武器名保存为可供后续逻辑使用的变量。
- 修正 CTI 搜索逻辑：输入关键词时执行“模糊搜索”而非仅别名匹配，支持武器名片段、首字母缩写与不完整缩写。
- CTI 抽屉示例能力：`赫拉芬格/赫拉/hlfg` 均可命中“赫拉芬格”；`lc` 命中“落草”；`l` 至少可命中“落草、理性告别”。

## Impact
- Affected specs: 配置面板交互、武器数据读取、前端状态管理
- Affected code: 配置界面组件、武器面板子组件、武器列表数据加载逻辑、样式文件

## ADDED Requirements
### Requirement: 武器选择入口与默认状态
系统 SHALL 在配置界面/武器面板提供一个可点击的武器选择方框，且默认显示“无”。

#### Scenario: 默认展示
- **WHEN** 用户首次进入该配置界面且尚未进行选择
- **THEN** 武器方框显示“无”
- **AND** 内部武器变量值为“无”或等价空值映射

### Requirement: 武器选择抽屉交互
系统 SHALL 在点击武器方框后打开向上抽屉，并显示可选武器列表。

#### Scenario: 打开抽屉
- **WHEN** 用户点击武器方框
- **THEN** 向上抽屉出现
- **AND** 抽屉内展示来自 `weapons-list.json` 的武器项与“无”选项

### Requirement: 悬停高亮反馈
系统 SHALL 为武器方框及抽屉选项提供统一的悬停高亮反馈。

#### Scenario: 方框悬停
- **WHEN** 鼠标悬停在武器方框上
- **THEN** 方框显示 `rgba(255, 255, 0, 0.3)` 高亮效果

#### Scenario: 选项悬停
- **WHEN** 鼠标悬停在抽屉任意武器选项上
- **THEN** 该选项显示 `rgba(255, 255, 0, 0.3)` 高亮效果

### Requirement: 选择结果回填与状态保存
系统 SHALL 在用户完成选择后即时更新方框文本并保存当前武器名变量。

#### Scenario: 选择具体武器
- **WHEN** 用户在抽屉中点击某个武器名
- **THEN** 方框显示对应武器名
- **AND** 当前武器变量更新为该武器名

#### Scenario: 选择“无”
- **WHEN** 用户在抽屉中点击“无”
- **THEN** 方框显示“无”
- **AND** 当前武器变量重置为“无”或等价空值映射

### Requirement: CTI 武器模糊搜索
系统 SHALL 在 CTI 文本输入框提供武器模糊搜索，并通过向上抽屉展示匹配项。

#### Scenario: 武器名片段匹配
- **WHEN** 用户在 CTI 输入 `赫拉芬格` 或 `赫拉`
- **THEN** 抽屉展示“赫拉芬格”候选项

#### Scenario: 缩写匹配
- **WHEN** 用户在 CTI 输入 `hlfg`
- **THEN** 抽屉展示“赫拉芬格”候选项

#### Scenario: 不完整缩写匹配
- **WHEN** 用户在 CTI 输入 `lc`
- **THEN** 抽屉展示“落草”候选项
- **WHEN** 用户在 CTI 输入 `l`
- **THEN** 抽屉至少展示“落草、理性告别”候选项

#### Scenario: CTI 抽屉选择同步
- **WHEN** 用户在 CTI 抽屉点击任意武器候选项
- **THEN** 当前武器变量更新为被选武器
- **AND** 武器方框文本同步显示被选武器名

## MODIFIED Requirements
### Requirement: 配置界面交互结构
系统 SHALL 在现有配置界面交互中同时支持“武器选择方框抽屉”与“CTI 输入模糊搜索抽屉”两条流程，使武器配置可在当前面板内完成，并保持两条流程状态一致。

## REMOVED Requirements
### Requirement: 无
**Reason**: 本次需求不移除既有功能，仅新增并扩展交互能力。  
**Migration**: 无迁移要求。
