而是一路推演下去。

  1. 01

    组成队伍

    从本地干员库选择出战角色，准备武器、装备与属性配置。

  2. 02

    编排时间轴

  3. 03

    保存为文档

    完整配置按内容保存，重要节点可成为可恢复的快照。

  4. 04

    审查 AI 提案

    AI 在隔离工作节点里探索；你可以检查差异、应用或丢弃。

WORK NODE TREE / REVIEW●

THE COMPLETE PROJECT FLOW

从选人开始，
到一份可追溯的结论。

这不是一组孤立页面。工作台把资料、配置、排轴、计算、版本和 AI 协作连成一条连续链路；每一段都可以回看，也可以回到前一段重新推演。

01选人建队→02角色配置→03排轴与 Buff→04伤害复核→05保存与恢复→06AI 工作节点→07验证与交付

01 → 02 / PREPARE

先确定“这是谁”，
再讨论“怎么打”。

从本地干员库组成最多四人的队伍；随后在角色配置页调整等级、潜能、主副能力、武器、装备与 A/B/E/Q 技能。面板、技能和配置同时可见，避免计算建立在模糊前提上。

  • 本地干员、武器、Buff 与自定义资料
  • 角色面板与技能等级配置
  • 图片资源的下载、导入与本地管理

03 → 04 / SIMULATE

把技能、状态与命中，
放回真正的时间线上。

在排轴画布安排技能按钮、拖动节点、设定敌方抗性，并为每一段命中按需加入或停用 Buff。技能详情可继续微调层数、异常状态和命中段，计算过程也能回看它们进入了哪个乘区。

  • 技能节点拖拽、复制、锁定与删除
  • 目标抗性、Hit 微调与 Buff 批量操作
  • 表格、伤害计算与报告输出
SKILL / HIT DETAIL计算不是黑箱。

状态、Buff、抗性和每段命中都可沿时间轴回看。

DMG · BUFF · RESISTANCE
技能节点异常状态伤害乘区

05 / PRESERVE

方案不是一次保存，
而是一份可恢复的记录。

排轴可以保存为不可变快照，也可以作为完整 SQLite 文档重新打开。恢复面板明确列出干员、技能按钮与 Buff 数量，让“回到哪一步”成为可见选择，而不是覆盖式操作。

  • 命名快照与恢复前的内容概览
  • 完整排轴文档、分享包与原子导入
  • 快照、工作节点、checkout 与审计事件

06 / REVIEW

AI 可以探索，
但每一次探索都有节点。

AI 的操作先进入隔离的 Work Node，而不是直接改写当前方案。节点树同时呈现草稿、校验、分支、风险、Diff 与 checkout / restore 证据；你可以据此审查、应用或丢弃。

  • 工作副本、补丁与差异检查
  • AI 草稿 / AI 校验 / 人工 checkpoint
  • 风险标记、回退点与可见审查链

07 / VERIFY & DELIVER

从一份本地方案，
到经得起回放的产品能力。

运行时的演进不只看一次演示是否成功。Harness 将规则、技能、路由与真实会话场景版本化，对候选行为进行回放与比较，关注修复是否有效、原有能力是否退化，以及用户状态是否安全。

队伍进入排轴后，
技能不再是一张静态表。

最多四名干员构成出战队列。排轴画布按分组组织 A / B / E / Q 技能节点；每个节点都能继续承载 Buff、状态、敌方抗性、命中微调和伤害详情。

四人出战队列技能节点拖拽与分组批量 Buff 操作目标抗性 / Hit 微调

02 / CALCULATE → REPORT

计算之后，
还要把过程讲清楚。

伤害报告将队伍配置、两组排轴与伤害图表放进同一份可读页面，展示总伤、角色占比和过程时序；底部的表格入口可继续把结果带出工作台。

LOCAL DATA STUDIO

资料不是只读素材；它们本身也是可维护的工作表。

干员、武器、装备和 Buff 都有独立的本地编辑面，可新建、保护、整理、保存、导入与导出。图片资源另有专门的管理器，而不是散落在文件夹里。

VERSION CONTROL FOR A TIMELINE

保存、恢复、分支与 AI 协作，
都不是隐藏在背景里的动作。

恢复面板列出每个快照的队伍、按钮和 Buff 数量；完整 SQLite 文档再提供工作节点树。AI 草稿、校验节点、人工 checkpoint、风险与回退路径一并可见。
