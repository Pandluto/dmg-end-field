Route requests through the existing native Workbench intent and knowledge flow. Native catalog evidence is a chosen retrieval tool, not a global turn route.

Only a request that judges which weapon or equipment better fits a specific operator—an operator-specific recommendation, optimization, `3+1` plan, or suitability comparison—begins with `def_data_operator_build_guide`, even without a named author. Pure catalog facts, field/ID/slot/effect lookups, and comparisons that do not judge operator fit use the narrowest trusted typed catalog resource and do not require guide discovery. The guide tool's exact-operator result is `GUIDE_FOUND`, `PARTIAL_GUIDE_FOUND`, or `GUIDE_NOT_FOUND`; never infer those states from a generic knowledge-search candidate. Guide strategy proceeds to catalog verification. Only partial/not-found results authorize `def_data_operator_build_profile`, using their exact fallback token. Source-only “攻略怎么说” remains a separate `def_data_game_knowledge` route.

Combat mechanisms, trigger chains, support-role utility, and “why this weapon works” use the separate `def_data_combat_conventions` branch only when guide/profile evidence marks it required. It returns reviewed condition rules, not a guide excerpt and not current product facts. A `GUIDE_FOUND` result marked `not-required` routes directly to `def_data_weapon_fit_plan` without a convention hash. A weapon-fit recommendation finishes with that planner, never a truncated weapon/loadout summary; a terminal planner error cannot route to a fallback tool family.

An exact named-guide read makes the guide team route sticky for that native session. Every later request to preview, correct, confirm, or apply that guide's team equipment must use the one team-plan family: plan, returned-decision revision, then one team apply. Never fall back to per-operator previews or patches, including after a slot error or after older per-operator proposal tokens exist. A correction rebuilds the unified guide plan; it does not split the team into four independent proposals.
