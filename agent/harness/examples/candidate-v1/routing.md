Route requests through the existing native Workbench intent and knowledge flow. Native catalog evidence is a chosen retrieval tool, not a global turn route.

Only a request that judges which weapon or equipment better fits a specific operator—an operator-specific recommendation, optimization, `3+1` plan, or suitability comparison—begins with `def_data_operator_build_guide`, even without a named author. Pure catalog facts, field/ID/slot/effect lookups, and comparisons that do not judge operator fit use the narrowest trusted typed catalog resource and do not require guide discovery. The guide tool's exact-operator result is `GUIDE_FOUND`, `PARTIAL_GUIDE_FOUND`, or `GUIDE_NOT_FOUND`; never infer those states from a generic knowledge-search candidate. Guide strategy proceeds to catalog verification. Only partial/not-found results authorize `def_data_operator_build_profile`, using their exact fallback token. Source-only “攻略怎么说” remains a separate `def_data_game_knowledge` route.

Combat mechanisms, trigger chains, support-role utility, and “why this weapon works” use the separate `def_data_combat_conventions` branch. It returns reviewed condition rules, not a guide excerpt and not current product facts. When guide discovery marks combat conventions required, resolve that bundle before the fallback profile and pass its exact hash unchanged. A weapon-fit recommendation finishes with `def_data_weapon_fit_plan`, never a truncated weapon/loadout summary.
