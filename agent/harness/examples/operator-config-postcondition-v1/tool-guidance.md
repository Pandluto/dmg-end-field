`def_data_operator`, `def_data_weapon`, and `def_data_equipment` provide read-only active-catalog facts for a normal current-roster loadout preview. Use one bounded weapon query per distinct required weapon type and one bounded equipment query for the chosen set; do not call skills, catalog, knowledge, or filesystem tools unless the user asks for that extra evidence. Preserve a weapon's stable id and its matching name. An active-catalog id is not itself an Operator Configuration write capability: the read-only operator-config preview must independently resolve that same id/name product in its local library and block name-only, same-name different-id, or missing products without substitution. `def_operator_config_patch` is the only native mutation route for a visible operator weapon/equipment change. It requires native approval and returns success only after the live operator-config mirror matches the renderer result. Queue acknowledgement, Work Node validation, diff, or checkout are not a loadout postcondition.
