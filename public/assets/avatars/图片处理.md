# avatars 图片处理 Skill

## 目标
在 `data/img/avatars/<干员名称>/` 下，自动识别每张图片所属类型，并按统一命名规范重命名。  
以阿列什目录中的以下四张图为标准样式：
- `阿列什.png`
- `阿列什战技.png`
- `阿列什连携技.png`
- `阿列什终结技.png`

## 适用范围
- 仅处理路径：`data/img/avatars/<干员名称>/`
- 仅处理扩展名：`.png`
- 仅重命名“可明确归类到四种类型”的文件

## 标准命名规范
对任意干员 `<N>`，目标文件名固定为：
- 角色头像：`<N>.png`
- 战技图标：`<N>战技.png`
- 连携技图标：`<N>连携技.png`
- 终结技图标：`<N>终结技.png`

## 分类判定规则
按文件名进行字符串模式匹配（推荐使用正则或关键字匹配）：

1. **角色头像（Avatar）**
- 命中条件（任一）：
- 文件名等于全大写英文名（如 `ANTAL.png`、`LIFENG.png`）
- 文件名已是中文名本体（如 `阿列什.png`）

2. **战技（Active Skill）**
- 命中条件（任一）：
- 文件名包含 `icon_skill_`
- 文件名包含 `战技`
- 排除条件：
- 同时包含 `ultimate`、`combo`、`talent` 时不归为战技

3. **连携技（Chain Skill）**
- 命中条件（任一）：
- 文件名包含 `icon_combo_skill_`
- 文件名包含 `连携技`

4. **终结技（Ultimate Skill）**
- 命中条件（任一）：
- 文件名包含 `icon_ultimate_skill_`
- 文件名包含 `终结技`

## 冲突与优先级
当一个文件名同时命中多个模式时，按以下优先级决策：
1. `icon_ultimate_skill_` -> 终结技
2. `icon_combo_skill_` -> 连携技
3. `icon_skill_` -> 战技
4. 头像规则

说明：该优先级用于避免 `skill` 被误判为普通战技而覆盖终结技/连携技。

## 重命名执行策略
1. 遍历 `data/img/avatars` 下每个干员目录。
2. 对目录内每个 `.png` 文件执行分类判定。
3. 若判定属于四类之一，则映射到标准文件名并执行重命名。
4. 若不属于四类任意一种（如 `icon_talent_*`、`icon_battle_*`、`*_line.png`、`magma_*.png`），**不做重命名**。
5. 若四类中有缺失项，**直接忽略缺失，不补图、不报错中断**。

## 幂等性要求
- 已符合目标命名的文件再次执行时应保持不变。
- 重复执行不应引入额外命名漂移。

## 安全约束
- 若目标文件名已存在，建议先做一致性校验（尺寸/哈希可选），避免误覆盖。
- 默认建议采用“先 dry-run 再 apply”：
- dry-run：仅输出 `old -> new`
- apply：确认后落地重命名

## 示例（阿列什）
- `阿列什.png` -> `阿列什.png`（保持）
- `阿列什战技.png` -> `阿列什战技.png`（保持）
- `阿列什连携技.png` -> `阿列什连携技.png`（保持）
- `阿列什终结技.png` -> `阿列什终结技.png`（保持）
- `icon_talent_deepfin_01.png` -> 不处理
- `icon_skill_deepfin_01_line.png` -> 不处理（非四类标准图）

## 示例（埃特拉）
目录：`data/img/avatars/埃特拉/`

- `ESTELLA.png` -> `埃特拉.png`（命中“角色头像”规则）
- `icon_skill_whiten_01.png` -> `埃特拉战技.png`（命中 `icon_skill_`）
- `icon_combo_skill_whiten_01.png` -> `埃特拉连携技.png`（命中 `icon_combo_skill_`）
- `icon_ultimate_skill_whiten_01.png` -> `埃特拉终结技.png`（命中 `icon_ultimate_skill_`）
- `icon_talent_whiten_01.png` -> 不处理（天赋图标，不属于四类）
- `icon_talent_whiten_02.png` -> 不处理（天赋图标，不属于四类）
